# math4rs v0.4 規劃書：線性代數模組 (linear_algebra/)

## 1. 目標

使用 Rust 實作矩陣/向量運算庫，與 math4js 的 linear_algebra 模組相容。基於 `Vec<Vec<f64>>` 而非 ndarray。

## 2. linear_algebra/ 模組結構

```
src/linear_algebra/
├── mod.rs              # 統一 export
├── matrix.rs           # Matrix 結構與方法
├── vector.rs           # 向量函數
├── decomposition.rs   # LU, QR, SVD, Cholesky 分解
├── eigensystem.rs     # 特徵值與特徵向量
├── linear_eq.rs       # 線性方程組求解
└── theorem.rs         # 線性代數定理驗證
```

## 3. API 設計（與 math4js 相容）

### 3.1 matrix.rs — Matrix 結構

```rust
pub struct Matrix {
    pub(crate) data: Vec<Vec<f64>>,
}
```

| API | 說明 |
|-----|------|
| `Matrix::new(data: Vec<Vec<f64>>) -> Matrix` | 建構矩陣 |
| `Matrix::zeros(rows, cols) -> Matrix` | 零矩陣 |
| `Matrix::ones(rows, cols) -> Matrix` | 全 1 矩陣 |
| `Matrix::identity(n) -> Matrix` | 單位矩陣 |
| `Matrix::eye(n) -> Matrix` | 單位矩陣（同 identity） |
| `Matrix::random(rows, cols) -> Matrix` | 隨機矩陣 |
| `Matrix::diag(v: &[f64]) -> Matrix` | 對角矩陣 |
| `Matrix::from_diag(v: &[f64]) -> Matrix` | 從向量建立對角矩陣 |
| `a.rows() -> usize` | 列數 |
| `a.cols() -> usize` | 行數 |
| `a.shape() -> (usize, usize)` | 形狀 |
| `a.get(i, j) -> f64` | 取得元素 |
| `a.transpose() -> Matrix` | 轉置 |
| `a.add(&other) -> Matrix` | 矩陣加法 |
| `a.sub(&other) -> Matrix` | 矩陣減法 |
| `a.scale(k) -> Matrix` | 純量乘法 |
| `a.matmul(&other) -> Matrix` | 矩陣乘法 |
| `a.dot(&other) -> Matrix` | 矩陣乘法（同 matmul） |
| `a.det() -> f64` | 行列式 |
| `a.inv() -> Result<Matrix, &str>` | 反矩陣 |
| `a.trace() -> f64` | 跡（對角線和） |
| `a.norm(ord) -> f64` | 矩陣範數 |
| `a.rank() -> usize` | 秩 |
| `a.row(i) -> Vec<f64>` | 取得第 i 列 |
| `a.col(j) -> Vec<f64>` | 取得第 j 行 |
| `a.to_vec() -> Vec<Vec<f64>>` | 轉換為 Vec |

## 4. 實作重點

### 4.1 Matrix vs NdArray

Linear algebra 的 Matrix 是 `Vec<Vec<f64>>`，ndarray 的 Array 是 `ndarray::ArrayD<f64>`。兩者是不同的類型，用於不同的場景。

### 4.2 特徵值計算

使用 QR 迭代法計算特徵值，收斂速度較慢但不需要外部庫。

## 5. 測試策略

- 矩陣運算正確性（加、減、乘、逆、行列式）
- 分解演算法正確性（LU, QR, SVD, Cholesky）
- 特徵值/特徵向量驗證
- 線性方程組求解驗證
