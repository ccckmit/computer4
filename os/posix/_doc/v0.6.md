# v0.6 — Phase 6: Shell

## Tool implemented

| Tool | Description |
|------|-------------|
| `sh` | Minimal POSIX shell with builtins and command execution |

## Features

- **Interactive REPL** (`$ ` prompt) when stdin is a terminal
- **Piped stdin mode** when input is piped (e.g., `echo "echo hi" | sh`)
- **Script mode**: `sh script.sh`
- **Builtins**: `cd`, `exit`, `export`, `echo`, `type`
- **Variable expansion**: `$VAR` in command arguments
- **PATH lookup** for external commands
- **Environment variable assignment** before commands: `VAR=value cmd`

## Design decisions

- Uses `std::io::stdin().is_terminal()` to distinguish interactive vs piped modes
- Tokenizer handles single quotes, double quotes, backslash escapes, and `$VAR` expansion
- Builtins are stored as function pointer tuples for dispatch
- External commands use `std::process::Command` with inherited stdio

## Known limitations

- No pipelines (`|`), no I/O redirection (`>`, `<`)
- No control flow (`if`, `for`, `while`, `case`)
- No job control (`&`, `jobs`, `fg`, `bg`)
- No functions, no aliases, no history
- `cd` does not support `OLDPWD` / `cd -`
- No error handling for most edge cases
