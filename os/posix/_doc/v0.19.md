# v0.19 — Shell 強化（重導向、Heredoc、順序執行、背景）

## 目標
將 `sh` 從簡易指令執行器升級為支援 POSIX shell 基本語法的直譯器，涵蓋 I/O 重導向、heredoc、指令順序、背景執行。

## 新增功能

### 1. I/O 重導向 (Redirection)
| 語法 | 說明 |
|------|------|
| `cmd > file` | stdout → file（覆寫） |
| `cmd >> file` | stdout → file（附加） |
| `cmd < file` | stdin ← file |
| `cmd 2> file` | stderr → file |
| `cmd 2>> file` | stderr → file（附加） |
| `cmd > file 2>&1` | stdout+stderr → file |

實作方式：在 `execute_line` / `execute_pipeline` 中，先從 token 序列中提取重導向配對（`op`, `target`），剩餘 token 為指令與引數。使用 `Stdio::from(File::create(...))` 設定 `child.stdout()` / `child.stderr()` / `child.stdin()`。

### 2. Heredoc (`<<`)
```
cmd <<EOF
content
EOF
```
將內嵌文字餵入指令的 stdin。實作方式：遇到 `<<` token 時，讀取後續行直到遇到分隔符（預設為 `EOF`，可使用任意字串）。若分隔符未加引號則進行變數擴展，加引號則逐字傳遞。

實作 `<<-`（tab 前綴忽略） 為進階選項。

### 3. 指令順序執行
| 語法 | 說明 |
|------|------|
| `cmd1 ; cmd2` | 依序執行 |
| `cmd1 && cmd2` | cmd1 成功後才執行 cmd2 |
| `cmd1 \|\| cmd2` | cmd1 失敗後才執行 cmd2 |

實作方式：在 `execute_line` 中先以 `;`、`&&`、`||` 分割，再個別執行，根據前一個指令的退出碼決定是否執行下一個。

### 4. 背景執行 (`&`)
```
cmd &
```
在背景啟動指令，不等待完成即顯示提示。印出 `[1] pid`。

實作方式：`child.spawn()` 後不呼叫 `.wait()`，直接印出 job 資訊。`wait` 內建指令可等待所有背景 job。

### 5. 變數擴展強化
- `$VAR` 與 `${VAR}` 語法
- `${VAR:-default}` 預設值
- `$?` 上一個指令退出碼

## 實作架構變更

```
execute_line(line)
  ├─ split_sequences(line) → [Seq(commands, op)]   # 依 ; && || 分割
  ├─ execute_sequence(cmds, op)
  │   ├─ parse_redirects(tokens) → (cmd_tokens, redirects)
  │   ├─ heredoc 處理（<< 先行讀取內容）
  │   ├─ tokenize → expand_vars → build_command
  │   └─ spawn / wait
  └─ job control（背景 &）
```

## Tokenizer 擴充
新增 token 類型：

| Token | 優先度 |
|-------|--------|
| `\|` | pipe |
| `;` | sequence |
| `&&` | and |
| `\|\|` | or |
| `&` | background |
| `<` `>` `>>` `<<` `2>` `2>>` | redirect |

## 測試計畫

| 測試 | 情境 |
|------|------|
| `test_sh_redirect_out` | `echo hello > /tmp/out.txt` |
| `test_sh_redirect_append` | `echo hello >> /tmp/out.txt` |
| `test_sh_redirect_in` | `wc -c < /tmp/in.txt` |
| `test_sh_heredoc` | `cat <<EOF\nhello\nEOF` |
| `test_sh_sequential` | `echo a; echo b` 輸出 a b |
| `test_sh_and` | `true && echo ok` |
| `test_sh_or` | `false || echo fallback` |
| `test_sh_background` | `sleep 0 &` 印出 job id |
| `test_sh_var` | `echo $HOME` |

## 相依性
無新增外部依賴 — 僅使用 `std::process::Stdio`、`std::fs::File`、`std::fs::read_to_string`。
