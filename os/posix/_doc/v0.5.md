# v0.5 — Phase 5: System Tools

## Tools implemented

| Tool | Options | Description |
|------|---------|-------------|
| `ps` | `-a` | Process status |
| `du` | `-s` `-h` `-d N` | Disk usage |
| `df` | `-h` | Disk free (stub) |
| `stty` | (none) | Terminal settings (stub) |
| `test` | `-n` `-z` `=` `!=` `-eq` `-ne` `-lt` `-le` `-gt` `-ge` `-e` `-f` `-d` `!` | Expression evaluation |
| `nice` | `-n N` | Run with adjusted niceness |
| `nohup` | (none) | Run immune to hangups |
| `date` | `-u` `+format` | Print/set date/time |

## Design decisions

- `test` (also available as `[`) supports string comparisons, integer comparisons, file tests, and `!` negation
- `date` uses `libc::localtime_r` for local timezone on Unix, `libc::gmtime_r` for UTC
- `date` format strings support `%Y` `%m` `%d` `%H` `%M` `%S`
- `ps -a` reads `/proc` entries on Linux
- `du` does recursive directory size calculation with opt-in human-readable format
- `nohup` redirects stdout to `nohup.out` (unconditionally), stdin to `/dev/null`

## Test coverage

95 integration tests. Debug + release: zero warnings.

## Known limitations

- `ps` only supports `/proc` on Linux; macOS fallback shows only current process
- `df` is a stub that only shows header and mount point name
- `stty` is a stub that prints hardcoded terminal settings
- `nice` does not properly set priority before exec on all platforms
- `nohup` always writes to `nohup.out` even if stdout is not a terminal
- `date` format with timezone info (%z, %Z) is not supported
