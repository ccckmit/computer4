# v0.9 — 檔案系統工具

## 新增工具

| 工具 | POSIX | 說明 | 關鍵實作 |
|------|-------|------|----------|
| `file` | 強制 | 判斷檔案類型 | Magic bytes 偵測（ELF, PNG, JPEG, GIF, PDF, PE, Mach-O, ZIP, gzip, bzip2, XZ）+ 文字/UTF-8 啟發式 |
| `chgrp` | 強制 | 變更群組所有權 | `libc::chown()` + `libc::getgrnam()` 群組名稱查詢 |
| `link` | XSI | 建立硬連結 | `libc::link()` |
| `unlink` | XSI | 移除檔案 | `libc::unlink()` |
| `mkfifo` | 強制 | 建立 FIFO | `libc::mkfifo()` 模式 0644 |
| `pathchk` | 強制 | 檢查路徑名稱 | 檢查空路徑、過長（>255）路徑、檔名內含 `/` |

## 實作細節

- `file` 內建 15+ 種魔術數字偵測，無外部依賴；針對 macOS Mach-O 萬用二進位與 64-bit 可執行檔有對應偵測
- `chgrp` 接受群組名稱（`getgrnam`）或數字 GID；需要 root 權限才能實際變更
- `link`/`unlink` 直接對應 POSIX syscall，是 `ln`/`rm` 的底層等效工具
- `mkfifo` 使用 `mkfifo()` syscall，不受 `umask` 影響（固定 0644）
- `pathchk` 驗證 `PATH_MAX`（255）限制，檢測空的與含 `/` 的檔名

## 整合測試

9 個新測試案例：file（目錄/文字/ELF/Mach-O）、chgrp（無引數錯誤處理）、link/unlink（配對測試）、mkfifo、pathchk（有效/空白/過長）
