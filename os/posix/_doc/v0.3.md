# v0.3 — Phase 3: Text Processing

## Tools implemented

| Tool | Options | Description |
|------|---------|-------------|
| `head` | `-n N` `-c N` | Output first part of files |
| `tail` | `-n N` `-c N` | Output last part of files |
| `sort` | `-r` `-n` `-u` | Sort lines of text |
| `uniq` | `-c` `-d` `-u` | Report/omit repeated lines |
| `cut` | `-d D` `-f LIST` `-c LIST` `-b LIST` | Cut selected portions of lines |
| `tr` | `-d` `-s` `-c` | Translate/delete characters |
| `tee` | `-a` | Read stdin, write to stdout + files |
| `od` | (none) | Octal dump |
| `cmp` | `-s` | Compare two files byte by byte |
| `diff` | (none) | Compare files line by line |

## Design decisions

- `sort` supports `-r` (reverse), `-n` (numeric), `-u` (unique); numeric uses `f64::parse`
- `uniq` implements `-c` (count), `-d` (repeated only), `-u` (unique only)
- `cut` handles field ranges with `-f`, delimiter with `-d`; `-c`/`-b` for character/byte ranges
- `tr` supports character expansion (`a-z`), `-d` delete, `-s` squeeze, `-c` complement
- `tee` buffers 8KB reads; writes to all output files and stdout simultaneously
- `od` outputs 16-bit word groups in octal, with address offset
- `cmp` byte-by-byte comparison with 1-indexed position reporting
- `diff` simple line-by-line comparison with unified-diff-style markers

## Known limitations

- `sort -n` uses `f64` which may lose precision for very large integers
- `diff` is a simple line-by-line comparator, not a full Myers/LCS algorithm
- `od` only does 16-bit word octal dump; no `-b`, `-c`, `-x`, `-j`, `-N` options
- `tr` set complement (`-c`) with `-d` is functional but may behave unexpectedly with multibyte UTF-8
- `cut` range syntax supports `N-M` open-ended (`N-` goes to end); max 1000 items per range
- `cmp` does not support `-l` (verbose offset list) or `-n` (byte limit)
- All tools are line-based (`BufRead`) and assume UTF-8 or ASCII; no multibyte character awareness
