# v0.2 — Phase 2: File Operations

## Tools implemented

| Tool | Options | Description |
|------|---------|-------------|
| `ls` | `-a` `-l` `-d` `-F` `-i` `-s` `-r` `-R` `-1` `-m` `-p` | List directory contents |
| `cp` | `-R` `-p` `-i` `-f` | Copy files and directories |
| `mv` | `-i` `-f` | Move/rename files |
| `rm` | `-r`/`-R` `-f` `-i` | Remove files/directories |
| `mkdir` | `-p` `-m mode` | Create directories |
| `rmdir` | `-p` | Remove empty directories |
| `ln` | `-s` `-f` | Create hard/soft links |
| `touch` | `-a` `-c` `-m` `-r ref` `-t time` | Update file timestamps |
| `chmod` | `-R` | Change file mode (octal + symbolic) |
| `chown` | `-R` | Change file owner/group |

## Design decisions

- Symbolic mode parsing (`u+rwx`, `go-w`, `a+r`) supported natively in `chmod`
- `ln` detects directory target and auto-places links inside it (POSIX convention)
- `touch` uses `libc::utimes` on Unix; `parse_time` handles `[[CC]YY]MMDDhhmm[.SS]` format
- `mkdir -p` delegates to `std::fs::create_dir_all`
- `rmdir -p` removes empty parent chain upward, stopping at `/`

## Test coverage

64 integration tests total (41 Phase 1 + 23 Phase 2). Release build: zero warnings.

## Known limitations

- `ls -R` prints directory headers but does not recursively descend into subdirectories
- `ls -l` does not show symlink targets
- `chown` typically requires root; test verifies it accepts syntax without crashing
- `touch -t` time parsing is basic; does not handle century offsets or negative timestamps
- All tools Unix-only (`#[cfg(unix)]` for `utimes`, `chown`, symlinks)
