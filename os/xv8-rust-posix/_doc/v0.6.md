# v0.6 — POSIX 行程管理 (Process Management)

> 新增 `setsid`、`getpgid`、`getppid`、`nice` 系統呼叫。

## 任務

- [x] **`sys_setsid`** — 建立新 session，行程成為 session leader + process group leader
- [x] **`sys_getpgid`** — 取得指定行程的 process group ID（pid=0 表示呼叫者自身）
- [x] **`sys_getppid`** — 取得父行程 PID（透過 `PROC_TABLE.parents` 陣列）
- [x] **`sys_nice`** — 調整行程優先權，值域 [-20, 19]
- [x] **`ProcInner` 擴充** — 新增 `pgid`、`sid`、`nice` 欄位
- [x] **fork 繼承** — 子行程繼承父行程的 `pgid` 與 `nice`
- [x] **使用者包裝層** — `setsid()`、`getpgid()`、`getppid()`、`nice()`
- [x] **`_posix_proc` 測試** — 6 項子測試

## 核心變更

### Syscall enum

```rust
Syscall::SetSid   = 43,  // pid_t setsid(void)
Syscall::GetPgid  = 44,  // pid_t getpgid(pid_t pid)
Syscall::GetPpid  = 45,  // pid_t getppid(void)
Syscall::Nice     = 46,  // int nice(int inc)
```

### ProcInner 新增欄位

```rust
pub struct ProcInner {
    // ...現有欄位...
    pub pgid: Pid,   // process group ID
    pub sid: Pid,    // session ID
    pub nice: i8,    // nice value (-20..19)
}
```

初始化值：`pgid = pid`、`sid = 0`、`nice = 0`。

### sys_setsid

- 若行程已是 `pgid == pid`（process group leader），回傳 `EPERM`
- 若行程已是 `sid == pid`（session leader），回傳 `EPERM`
- 設定 `pgid = pid`、`sid = pid`

### sys_getpgid

- `pid == 0` 時回傳呼叫者自身的 pgid
- 遍歷 `PROC_TABLE` 尋找符合 pid 的行程
- 找不到回傳 `ESRCH`

### sys_getppid

- 透過 `PROC_TABLE.parents[child.id]` 取得父行程 PID
- 父行程死亡時回傳 1（init 收養後）

### sys_nice

- 原始值 `inner.nice + inc`，夾至 [-20, 19]
- 回傳值為新 nice 值（可能有正負）
- 因回傳值可為負，核心偏移 +20 返回，使用者空間 -20 還原，避免與錯誤碼混淆
