# v0.9 — Termios / M7

> 實作 `termios` 終端機控制結構，透過 `ioctl(TCGETS/TCSETS)` 存取，為未來標準終端機行為（ICANON、ECHO、ISIG）奠定基礎。

## 任務

- [x] **`Termios` 結構** — 定義於 ABI，`#[repr(C)]`，含 `c_iflag`/`c_oflag`/`c_cflag`/`c_lflag`/`c_cc[NCCS]`
- [x] **`TCGETS` / `TCSETS` 常數** — Linux 相容 ioctl 編號（`0x5401` / `0x5402`）
- [x] **`Console::termios` 欄位** — 儲存目前終端機屬性，初始化為全零
- [x] **`Console::ioctl` 處理** — `TCGETS` 拷貝至使用者、`TCSETS` 從使用者讀入
- [x] **使用者原始包裝** — `raw::tcgetattr(fd, *mut Termios)` / `raw::tcsetattr(fd, *const Termios)`
- [x] **使用者安全包裝** — `tcgetattr(Fd, &mut Termios)` / `tcsetattr(Fd, &Termios)`
- [x] **`_posix_tty` 測試** — 3 項子測試

## 核心變更

### Termios 結構（ABI）

`kernel/src/abi.rs:29-39`：

```rust
pub const NCCS: usize = 32;

#[repr(C)]
#[derive(Debug, Clone, Copy)]
pub struct Termios {
    pub c_iflag: u32,
    pub c_oflag: u32,
    pub c_cflag: u32,
    pub c_lflag: u32,
    pub c_cc: [u8; NCCS],
}
```

透過 `pub use kernel::abi::*;` 匯出至使用者空間，確保跨核心/使用者 ABI 一致。

### Ioctl 常數

`kernel/src/file.rs:358-359`：

```rust
impl Ioctl {
    pub const TCGETS: usize = 0x5401;
    pub const TCSETS: usize = 0x5402;
}
```

與 Linux 上游 ioctl 編號一致（`asm-generic/ioctls.h`）。

### Console::ioctl 實作

`kernel/src/console.rs:248-288`：

```rust
Ioctl::TCGETS => {
    let ptr = &console.termios as *const Termios as *const u8;
    let bytes = unsafe { core::slice::from_raw_parts(ptr, size_of::<Termios>()) };
    proc::copy_to_user(bytes, VA::from(arg)).map_err(|_| Errno::EFAULT)?;
    Ok(0)
}
Ioctl::TCSETS => {
    let mut bytes = [0u8; size_of::<Termios>()];
    proc::copy_from_user(VA::from(arg), &mut bytes).map_err(|_| Errno::EFAULT)?;
    console.termios = unsafe { core::ptr::read(bytes.as_ptr() as *const Termios) };
    Ok(0)
}
```

沿用既有 `CONSOLE.lock()` 模式，持有鎖期間進行拷貝以確保原子性。

### File::ioctl 分派

`kernel/src/file.rs:284-303` — 裝置型別為 `CONSOLE` 時轉發至 `Console::ioctl()`：

```rust
FileType::Device { major, .. } if *major as usize == CONSOLE => {
    Console::ioctl(cmd, arg)
}
```

### 使用者包裝層

`user/src/syscall.rs:190-196`（原始 API）：

```rust
pub fn tcgetattr(fd: usize, termios: *mut Termios) -> isize {
    syscall3(Syscall::Ioctl, fd, Ioctl::TCGETS, termios as usize)
}
pub fn tcsetattr(fd: usize, termios: *const Termios) -> isize {
    syscall3(Syscall::Ioctl, fd, Ioctl::TCSETS, termios as usize)
}
```

`user/src/syscall.rs:766-774`（安全 API）：

```rust
pub fn tcgetattr(fd: Fd, termios: &mut Termios) -> Result<(), Errno> {
    check_unit(raw::tcgetattr(fd.as_raw(), termios as *mut Termios))
}
pub fn tcsetattr(fd: Fd, termios: &Termios) -> Result<(), Errno> {
    check_unit(raw::tcsetattr(fd.as_raw(), termios as *const Termios))
}
```

## 設計考量

- **透過 ioctl 而非專用 syscall**：符合 POSIX/Linux 慣例，減少系統呼叫編號膨脹
- **Termios 定義於 `abi.rs`**：作為純資料結構（無方法），置於核心/使用者共享的 ABI 模組，確保 `#[repr(C)]` 佈局一致
- **Console 持有鎖期間拷貝**：避免 `copy_to_user`/`copy_from_user` 期間的競爭條件
- **全零初始值**：目前所有旗標位元不具備實際作用（raw 模式仍由 `Console::raw` bool 控制），預留給未來實作

## 已知限制

- `c_lflag` 的 ICANON / ECHO / ISIG 等旗標尚未被 Console 中斷處理器判讀 — raw/cooked 模式仍由 `Console::raw` bool 控制
- 無 `TCSETSW` / `TCSETSF` 變體（`TCSETS` 直接套用，無清空佇列動作）
- 無 `cfsetospeed` / `cfsetispeed` 等輔助函式（c_cflag 的 baud rate 為存根）
- 無 `winsize` 結構或 `TIOCGWINSZ` ioctl
- `NCCS = 32` 但 `c_cc` 陣列索引常數（VINTR / VERASE / VKILL 等）尚未定義
- 未驗證寫入權限（目前任何行程都可對 console fd 呼叫 `TCSETS`）
