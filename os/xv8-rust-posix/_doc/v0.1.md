# v0.1 — 基礎移植

> 從 xv7-rust-octopus 複製，更名為 xv8，核心可啟動並通過 xv7 既有測試。

## 變更清單

### 1. QEMU 相容性修正

在 Apple Silicon (arm64) 上 QEMU 10.2.2 需要 `-cpu max` 才能支援 RISC-V `menvcfg`/`mseccfg` CSR：

**`.cargo/config.toml`** — runner 加入 `-cpu max`：
```toml
[target.riscv64gc-unknown-none-elf]
runner = "qemu-system-riscv64 -machine virt -cpu max -bios none -m 128M -kernel"
```

### 2. 核心 crate 更名

所有套件從 `octopos` → `xv8`：
| 套件 | 原名稱 | 新名稱 |
|------|--------|--------|
| kernel | `octopos` 核心 | `xv8` 核心 |
| user | `user` | `user`（保留） |
| mkfs | `mkfs` | `mkfs`（保留） |

### 3. 開機訊息更新

**`kernel/src/lib.rs:57`** — 核心訊息改為 `"xv8 kernel is booting"`：
```rust
println!("xv8 kernel is booting");
```

### 4. 測試字串更新

`user/testbin/` 中各測試的 payload 字串從 `octopos` 改為 `xv8`。

### 5. 文件

- 新增 `_doc/plan.md`（里程碑規劃）
- 新增 `_doc/posix.md`（POSIX API 完整清單）
- 新增 `_doc/version.md`（版本路線圖）
- `README.md` 改為 xv8 標題

### 6. 專案文件

- `Cargo.lock` 重新產生（crate 更名後）
- `LICENSE` 保留 MIT
- `rust-toolchain.toml` 鎖定 nightly
