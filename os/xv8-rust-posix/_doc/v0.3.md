# v0.3 — 信號 (Signals)

> 核心信號框架，支援 SIGKILL/SIGTERM/SIGINT 等。

## 信號常數

```rust
pub const NSIG: usize = 32;

pub enum Signal {
    SIGHUP  = 1,   // Hangup
    SIGINT  = 2,   // Interrupt (Ctrl-C)
    SIGQUIT = 3,   // Quit
    SIGILL  = 4,   // Illegal instruction
    SIGTRAP = 5,   // Trace/breakpoint
    SIGABRT = 6,   // Abort
    SIGBUS  = 7,   // Bus error
    SIGFPE  = 8,   // Floating point exception
    SIGKILL = 9,   // Kill (cannot be caught/ignored)
    SIGUSR1 = 10,  // User-defined 1
    SIGSEGV = 11,  // Segmentation fault
    SIGUSR2 = 12,  // User-defined 2
    SIGPIPE = 13,  // Broken pipe
    SIGALRM = 14,  // Alarm clock
    SIGTERM = 15,  // Termination
    SIGCHLD = 17,  // Child status changed
    SIGCONT = 18,  // Continue
    SIGSTOP = 19,  // Stop (cannot be caught/ignored)
    SIGTSTP = 20,  // Keyboard stop
}
```

## 核心結構變更

### ProcInner 新增欄位

```rust
pub struct ProcInner {
    // ...現有欄位...
    pub sigactions: [SigAction; NSIG],  // 信號處理器表
    pub pending: u32,                   // 擱置信號 bitmask
    pub blocked: u32,                   // 遮罩信號 bitmask
}

pub struct SigAction {
    pub handler: usize,  // SIG_DFL=0, SIG_IGN=1, or function pointer
    pub flags: u32,
    pub mask: u32,
}
```

### 預設動作表

| 信號 | 預設動作 |
|------|----------|
| SIGKILL (9) | Terminate |
| SIGSTOP (19) | Stop |
| SIGCHLD (17), SIGCONT (18) | Ignore |
| 其餘 | Terminate |

## 信號遞送機制

在 `usertrap()` 返回使用者空間前：

1. 檢查 `pending & ~blocked`
2. 若無擱置信號 → 正常返回使用者空間
3. 若有信號：
   - **SIGKILL / 預設動作 = terminate** → `proc::exit(-1)`
   - **SIGSTOP / 預設動作 = stop** → 行程進入 `Sleeping` 直到收到 SIGCONT
   - **SIG_IGN** → 清除 pending bit，繼續
   - **有自訂 handler** → 在使用者堆疊上建立 sigframe，設定 sepc 為 handler 位址
   - 清除 pending bit

### sigframe 結構

```rust
struct SigFrame {
    return_addr: usize,  // sa_sigaction 返回位址（sigreturn trampoline）
    epc: usize,          // 原始 sepc
    ra: usize,
    a0: usize,           // sigaction 的 arg
    // ... 其他需要保存的暫存器
}
```
