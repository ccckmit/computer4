# v0.7 — POSIX 時間 (Time)

> 新增 `clock_gettime` 與 `nanosleep` 系統呼叫，支援高精度時間查詢與休眠。

## 任務

- [x] **`sys_clock_gettime`** — CLOCK_REALTIME / CLOCK_MONOTONIC，回傳 `Timespec`
- [x] **`sys_nanosleep`** — 奈秒精度休眠，支援信號中斷回寫剩餘時間
- [x] **`Timespec` 結構** — `{ tv_sec: i64, tv_nsec: i64 }`，納入 ABI
- [x] **`CLOCK_REALTIME` / `CLOCK_MONOTONIC` 常數**
- [x] **使用者包裝層** — `clock_gettime()`、`nanosleep()`
- [x] **`_posix_time` 測試** — 5 項子測試

## 核心變更

### Syscall enum

```rust
Syscall::ClockGetTime = 41,  // int clock_gettime(clockid_t clock_id, struct timespec *tp)
Syscall::NanoSleep   = 42,   // int nanosleep(const struct timespec *req, struct timespec *rem)
```

### Timespec (ABI)

```rust
#[repr(C)]
#[derive(Debug, Clone, Copy)]
pub struct Timespec {
    pub tv_sec: i64,
    pub tv_nsec: i64,
}

pub const CLOCK_REALTIME: u32 = 0;
pub const CLOCK_MONOTONIC: u32 = 1;
```

### 時間基準

xv8 的時鐘中斷頻率約為 100ms/tick：

```rust
const NSEC_PER_TICK: i64 = 100_000_000;  // 100ms
const NSEC_PER_SEC: i64  = 1_000_000_000;
```

### sys_clock_gettime 實作

- 支援 `CLOCK_REALTIME` (0) 與 `CLOCK_MONOTONIC` (1)
- 讀取 `TICKS` 全域計數器，轉換為秒 + 奈秒
- 目前 `CLOCK_REALTIME` 起點同 `CLOCK_MONOTONIC`（開機至今）
- 無效 clock_id 回傳 `EINVAL`
- 透過 `copy_to_user` 將 `Timespec` 寫回使用者空間

### sys_nanosleep 實作

- 從使用者空間讀取 `req` Timespec（`copy_from_user`）
- 驗證 `tv_sec >= 0`、`tv_nsec >= 0`、`tv_nsec < 1e9`
- 計算所需 ticks = total_ns / NSEC_PER_TICK
- 以 `TICKS` channel 休眠（同 `sys_sleep` 機制）
- 若被信號中斷：
  - 計算剩餘時間寫回 `rem`（若 `rem` 不為 NULL）
  - 回傳 `EINTR`

## 已知限制

- `CLOCK_REALTIME` 尚未與真實 wall clock 同步（起點同開機）
- `clock_getres`、`clock_settime`、`timer_create` 等尚未實作
- `nanosleep` 最小精度受限於中斷頻率（100ms）
- 剩餘時間回寫 (`rem`) 使用 ticks 轉換，可能因中斷頻率取整而有誤差
