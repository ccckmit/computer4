# v0.5 — 記憶體映射 (Memory Mapping)

> 新增 `mmap` / `munmap` / `mprotect` 系統呼叫，支援匿名映射。

## 任務

- [x] **`sys_mmap`** — MAP_ANONYMOUS | MAP_PRIVATE，prot PROT_READ/WRITE/EXEC，位址自動選取
- [x] **`sys_munmap`** — 解除映射，釋放實體記憶體與頁表項目
- [x] **`sys_mprotect`** — 修改已映射區域的 Sv39 頁面權限
- [x] **`MmapRegion` 結構** — 記憶體映射中繼資料（start, len, prot, flags）
- [x] **`ProcData.mmap_regions`** — 行程的映射串列，fork 時繼承
- [x] **頁表內部可見性提升** — `PageTableEntry`、`walk_mut` 等改為 `pub(crate)`
- [x] **系統呼叫重新編號** — `Access` 從 34→48，釋出 34-36 給 Mmap/Munmap/Mprotect
- [x] **`syscall6` 使用者包裝** — 支援 6 引數系統呼叫（mmap 需要 addr+length+prot+flags+fd+offset）
- [x] **使用者常量** — `PROT_READ`/`PROT_WRITE`/`PROT_EXEC`/`PROT_NONE`、`MAP_ANONYMOUS`/`MAP_PRIVATE`/`MAP_FAILED`
- [x] **`_posix_mmap` 測試** — 5 項子測試

## 核心變更

### Syscall enum

```rust
Syscall::Mmap    = 34,  // void *mmap(void *addr, size_t length, int prot, int flags, int fd, off_t offset)
Syscall::Munmap  = 35,  // int munmap(void *addr, size_t length)
Syscall::Mprotect = 36, // int mprotect(void *addr, size_t len, int prot)
// Access 移至 48
Syscall::Access  = 48,
```

### MmapRegion

```rust
pub struct MmapRegion {
    pub start: VA,
    pub len: usize,
    pub prot: usize,
    pub flags: usize,
}
```

### mmap 實作策略

- 僅支援 `MAP_ANONYMOUS | MAP_PRIVATE`，其他旗標回傳 `ENOSYS`
- `PROT_NONE` 回傳 `EINVAL`（xv8 暫不支援無權限映射）
- 位址從 `data.size` 開始配置（類似 sbrk 向上成長）
- 頁面權限直接對應 Sv39 `PTE_R`/`PTE_W`/`PTE_X`
- 每次 `mmap` 立即配置實體頁面（eager allocation，非 lazy）

### munmap 實作策略

- 遍歷頁表找到對應頁面，寫入 0 釋放頁表項目
- 呼叫 `kfree` 歸還實體頁面
- 從 `mmap_regions` 移除符合的 `MmapRegion`

### mprotect 實作策略

- 遍歷頁表，重新寫入 PTE 權限位元
- 更新 `mmap_regions` 中對應項目的 `prot`
