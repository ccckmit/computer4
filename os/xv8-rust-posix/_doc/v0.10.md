# v0.10 — VFS + Mount

## 概述

v0.10 實現虛擬檔案系統（VFS）抽象層，支援多種類型的檔案系統掛載。

## 新增功能

### VFS 核心架構 (`kernel/src/vfs.rs`)

- `VfsOps` trait：定義虛擬檔案系統的操作介面
  - `name()` — 檔案系統名稱
  - `lookup(parent, name)` — 查詢目錄項目
  - `read(id, offset, dst)` — 讀取檔案內容
  - `write(id, offset, src)` — 寫入檔案內容
  - `readdir(id, cb)` — 讀取目錄項目
  - `stat(id)` — 取得檔案狀態

- `Mount` 結構：維護掛載點資訊
  - `prefix` — 掛載路徑前綴
  - `fs` — 檔案系統實作（`&'static dyn VfsOps`）
  - `root_id` — 檔案系統根目錄 ID

- `MOUNTS` 全域掛載表（MAX_MOUNTS=8）
- `mount(path, fs, root_id)` — 新增掛載點
- `umount(path)` — 移除掛載點
- `check_mount(resolved)` — 檢查路徑是否為掛載點
- `find_mount(path)` — 根據確切路徑查找掛載點
- `init()` — 初始化 VFS，設定 `VFS_CHECK_MOUNT` hook

### ProcFs (`/proc/self/status`)

- `ProcFs` 實作 `VfsOps` trait
- 提供 `/proc/self/status` 虛擬檔案
- 內容格式：`Name:\tprocfs\nPid:\t<pid>\nUid:\t0\nGid:\t0\n`
- 目錄結構：
  - `/` (ID=1) — 根目錄
  - `/self` (ID=2) — 目錄
  - `/self/status` (ID=3) — 檔案

### DevFs (`/dev/null`, `/dev/zero`)

- `DevFs` 實作 `VfsOps` trait
- `/dev/null` — 讀取返回 0 位元組，寫入忽略
- `/dev/zero` — 讀取返回 null 位元組，寫入忽略
- 目錄結構：
  - `/` (ID=1) — 根目錄
  - `/null` (ID=2) — 字元裝置
  - `/zero` (ID=3) — 字元裝置

### 路徑解析整合 (`kernel/src/fs.rs`)

- `InodeInner` 新增欄位：
  - `vfs: Option<&'static dyn VfsOps>` — VFS 實作指標
  - `vfs_id: u64` — VFS 內部 ID

- `Inode::alloc_vfs(vfs, vfs_id, type)` — 分配 VFS inode

- `Path::resolve_inner` 修改：
  - 追蹤累計路徑（`accumulated` 字串）
  - 每個元件查詢前檢查 `VFS_CHECK_MOUNT` hook
  - 若路徑觸發掛載點，切換至 VFS 查詢
  - 若 inode 已有 `vfs` 設定，使用 VFS 查詢剩餘路徑

### 系統呼叫

- `sys_mount` (80) — 掛載檔案系統
  - 參數：source, target, fstype, flags
  - 目前支援："/proc" → ProcFs, "/dev" → DevFs
  - 其他路徑返回 `EINVAL`

- `sys_umount` (81) — 卸載掛載點
  - 參數：target path
  - 從 MOUNTS 表移除該前綴的掛載點

### 使用者空間介面 (`user/src/syscall.rs`)

- 新增 `syscall4()` — 4 參數系統呼叫包裝
- 新增 `raw::mount(source, target, fstype, flags)` — 直接呼叫 sys_mount
- 新增 `raw::umount(target)` — 直接呼叫 sys_umount

## 測試

### `_posix_vfs` 測試（6 個子測試）

1. `mount /proc` — 掛載 ProcFs
2. `mount /dev` — 掛載 DevFs
3. `read /proc/self/status` — 讀取 procfs 內容，驗證包含 "procfs" 和 "Pid:"
4. `open /dev` — 開啟 DevFs 根目錄
5. `umount /proc` — 卸載 ProcFs
6. `umount /dev` — 卸載 DevFs

## 技術細節

### VFS inode 配置

虛擬檔案系統 inode 不來自磁碟，使用專門的 `Inode::alloc_vfs()` 配置：
- `dev=0`（區分原生磁碟 inode）
- `inum=0`（非磁碟 inode）
- `ref=1`（初始引用計數）
- `valid=true`（已初始化）

### 路徑解析流程

1. 解析絕對路徑或相對路徑，取得起始 inode
2. 對每個路徑元件：
   - 累計路徑字串
   - 嘗試 `VFS_CHECK_MOUNT` 查詢掛載表
   - 若命中掛載點，使用 VFS 查詢剩餘路徑
   - 否則使用傳統 `Directory::lookup()` 查詢磁碟
3. 若 inode 已設定 `vfs` 指標，強制使用 VFS

### 掛載表查詢

- `VFS_CHECK_MOUNT: OnceLock<fn(&str) -> Option<(&'static dyn VfsOps, u64)>>`
- 由 `vfs::init()` 在系統啟動時設定
- `check_mount()` 遍歷 MOUNTS 表，比對累計路徑與掛載前綴

## 已知限制

- 掛載點僅支援確切前綴匹配，不支援子目錄掛載
- `sys_mount` 的 source/fstype/flags 參數被忽略
- VFS inode 的 uid/gid 固定為 0
- 無法在掛載點下再掛載（不支援嵌套掛載）
