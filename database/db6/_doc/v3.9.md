# v3.9 - SQL + JSON Path + Index 整合

## 日期：2026-05-17

## 目標

SQL 語法支援 JSON Path 過濾條件，並自動使用 secondary index 加速查詢，與 query/ API 行為一致。

## 背景

### v3.8 的 JSON Path 查詢

```rust
// query/ API 已經支援 JSON Path
db.select("*")
    .from("users")
    .filter("$.age > 25")
    .execute();
```

### SQL 無法使用相同語法

```sql
-- 之前 SQL 不能用 JSON Path
SELECT * FROM users WHERE $.age > 25;  -- 語法錯誤

-- 必須用傳統欄位（但沒有結構化欄位）
SELECT * FROM users WHERE value LIKE '%age:30%';  -- 無法高效查詢
```

### v3.9 目標：SQL 支援 JSON Path

```sql
-- 現在 SQL 也可以使用 JSON Path 了
SELECT * FROM users WHERE @.age > 25;
SELECT * FROM users WHERE @.name = 'Alice';
SELECT * FROM users WHERE @.city = '台北' AND @.age > 30;
```

## 語法設計

### JSON Path 語法

SQL 中使用 `@` 符號代表 JSON 文件：

| SQL 語法 | 說明 |
|----------|------|
| `@.field` | 取得欄位值 |
| `@.nested.field` | 取得巢狀欄位 |
| `@.array[0]` | 取得陣列元素 |
| `@.field > 25` | 比較運算 |
| `@.name IN ('Alice', 'Bob')` | IN 運算 |

### 支援的運算子

```sql
-- 比較運算
SELECT * FROM users WHERE @.age > 25;
SELECT * FROM users WHERE @.name = 'Alice';
SELECT * FROM users WHERE @.age >= 18 AND @.age <= 65;
SELECT * FROM users WHERE @.name != 'Admin';

-- LIKE 運算（字串開頭）
SELECT * FROM users WHERE @.name LIKE 'A%';

-- IN 運算
SELECT * FROM users WHERE @.city IN ('台北', '台中', '高雄');

-- IS NULL / IS NOT NULL
SELECT * FROM users WHERE @.phone IS NULL;
SELECT * FROM users WHERE @.email IS NOT NULL;

-- 複合條件
SELECT * FROM users WHERE @.age > 25 AND @.city = '台北';
SELECT * FROM users WHERE @.age < 20 OR @.vip = true;
```

## API 設計

### SQL 語法

```sql
-- 基本查詢
SELECT * FROM users WHERE @.age > 25;

-- 指定欄位
SELECT @.name, @.age FROM users WHERE @.age > 25;

-- 插入 JSON 資料
INSERT INTO users VALUES ('key1', '{"name":"Alice","age":30,"city":"台北"}');

-- 更新 JSON 欄位（使用 JSON Path）
UPDATE users SET @.age = 31 WHERE @.name = 'Alice';

-- 刪除滿足條件的記錄
DELETE FROM users WHERE @.age < 18;
```

### Rust API 對照

| query/ API | SQL 等價 |
|------------|----------|
| `.filter("$.age > 25")` | `WHERE @.age > 25` |
| `.filter("$.name = 'Alice'")` | `WHERE @.name = 'Alice'` |
| `.filter("$.city IN ('台北','台中')")` | `WHERE @.city IN ('台北','台中')` |

### 建立索引

```sql
-- 在 JSON 欄位上建立索引
CREATE INDEX ON users(@.age);
CREATE INDEX ON users(@.city);

-- 檢視索引
SHOW INDEX FROM users;
```


### 刪除索引

```sql

```

### 查詢流程

```

         │
         ▼
┌─────────────────┐
│  Parser         │ 解析 @.age > 25
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Planner        │
│  ├─ 提取 path: @.age │
│  ├─ 檢查是否有索引  │
│  └─ 生成執行計畫    │
└────────┬────────┘
         │
    ┌────┴────┐
    │ 有索引   │ 無索引
    ▼         ▼
┌────────┐  ┌──────────┐
│ Index  │  │ Full Scan │
│ Seek   │  │ Filter    │
└────┬───┘  └────┬─────┘
     │            │
     └─────┬──────┘
           │
           ▼
     ┌─────────┐
     │ Result  │
     └─────────┘
```

### Planner 優化

```rust
fn plan_select(select: &SelectStmt) -> Result<Plan> {
    // 1. 解析 WHERE 條件中的 JSON Path
    if let Some(where_expr) = &select.where_expr {
        let json_paths = extract_json_paths(where_expr);
        
        // 2. 檢查每個 path 是否有索引
        for path in &json_paths {
            if let Some(index) = find_index(table, path) {
                // 使用索引計畫
                return plan_index_scan(table, index, where_expr);
            }
        }
    }
    
    // 3. 沒有索引，使用全表掃描 + filter
    plan_table_scan(select)
}
```

## 實作方式

### 1. AST 新增 JSON Path 表達式

```rust
// src/sql/parser/ast.rs

pub enum Expr {
    // ... 現有表達式 ...
    
    /// JSON Path 表達式，如 @.age > 25
    JsonPath {
        path: Vec<String>,      // ["age"] 或 ["city", "name"]
        op: CompareOp,
        value: Value,
    },
}

pub enum Value {
    // ... 現有類型 ...
    
    String(String),
    Integer(i64),
    Float(f64),
    Bool(bool),
    Null,
}

pub enum CompareOp {
    Eq,      // =
    Ne,      // !=
    Lt,      // <
    Le,      // <=
    Gt,      // >
    Ge,      // >=
    Like,    // LIKE
    In,      // IN (...)
    IsNull,   // IS NULL
    IsNotNull, // IS NOT NULL
}
```

### 2. Lexer 支援 @ 符號

### 3. Parser 解析 JSON Path

```rust
// src/sql/parser/parser.rs

fn parse_expr(&mut self) -> Result<Expr> {
    match self.peek() {
        Token::At => {
            self.advance();  // 消耗 @
            self.parse_json_path_expr()
        }
        // ...
    }
}

fn parse_json_path_expr(&mut self) -> Result<Expr> {
    // 消耗點號
    self.expect(Token::Dot)?;
    
    // 解析欄位名
    let field = self.expect_ident()?;
    let mut path = vec![field];
    
    // 處理巢狀欄位 (如 @.address.city)
    while self.peek() == Token::Dot {
        self.advance();
        path.push(self.expect_ident()?);
    }
    
    // 解析運算子和值
    let op = self.parse_compare_op()?;
    let value = self.parse_value()?;
    
    Ok(Expr::JsonPath { path, op, value })
}
```

### 4. Planner 產生索引計畫

```rust
// src/sql/planner/planner.rs

pub enum Plan {
    // ... 現有計畫 ...
    
    IndexScan {
        table: String,
        index_name: String,
        path: Vec<String>,
        condition: Expr,
    },
}

impl Planner {
    fn plan_where(&mut self, expr: &Expr) -> Result<Plan> {
        match expr {
            Expr::JsonPath { path, op, value } => {
                // 檢查是否有可用索引
                if let Some(index) = self.find_index_for_path(path) {
                    Ok(Plan::IndexScan {
                        table: self.current_table.clone(),
                        index_name: index.name.clone(),
                        path: path.clone(),
                        condition: expr.clone(),
                    })
                } else {
                    // 回退到全表掃描
                    Ok(Plan::TableScan { ... })
                }
            }
            // ...
        }
    }
}
```

### 5. Executor 支援 IndexScan

```rust
// src/sql/executor/executor.rs

fn execute_plan(&mut self, plan: &Plan) -> Result<ResultSet> {
    match plan {
        Plan::IndexScan { table, index_name, path, condition } => {
            // 1. 使用索引獲取候選 keys
            let candidates = self.engine.index_lookup(table, index_name, condition)?;
            
            // 2. 讀取候選 keys 的實際資料
            let mut rows = Vec::new();
            for key in candidates {
                if let Some(value) = self.engine.get(table_id, &key)? {
                    // 驗證完整條件（過濾器）
                    if eval_condition(&value, condition) {
                        rows.push((key, value));
                    }
                }
            }
            
            Ok(self.rows_to_result(rows))
        }
        // ...
    }
}
```

## 與 query/ API 的整合

### 統一的查詢最佳化

```rust
// 兩種 API 最終使用相同的執行路徑

// query/ API
db.select("*")
    .from("users")
    .filter("$.age > 25")
    .execute();

// SQL API
db.execute("SELECT * FROM users WHERE @.age > 25");

// 兩者都會：
// 1. 檢查是否有 age 索引
// 2. 使用 IndexSeek 或 TableScan
// 3. 返回相同結果
```

### 共享 Index 管理

```rust
// 索引是共享的，一處建立處處生效

// 用 query/ 建立
db.create_index("users", "age").unwrap();

// 用 SQL 查詢（自動使用索引）
db.execute("SELECT * FROM users WHERE @.age > 25").unwrap();

// 用 SQL 建立
db.execute("CREATE INDEX ON users(@.city)").unwrap();

// 用 query/ 查詢（自動使用新索引）
db.select("*")
    .from("users")
    .filter("$.city = '台北'")
    .execute();
```

```rust
fn test_sql_uses_index() {
```

## 不做的事項

- ❌ JSON Path 完全相容性（只支援基本路徑和比較）
- ❌ 陣列索引（如 `@.items[0]`）
- ❌ JSON Path 函數（如 `@.name || @.age`）
- ❌ 聚合查詢（GROUP BY, HAVING）
- ❌ JOIN（跨表關聯）

- JOIN 支援
