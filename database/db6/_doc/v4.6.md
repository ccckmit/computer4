## API 設計

### RetryConfig

```rust
#[derive(Debug, Clone)]
pub struct RetryConfig {
    pub max_retries: u32,          // 最大重試次數，預設 3
    pub initial_delay_ms: u64,     // 初始延遲（毫秒），預設 100
    pub max_delay_ms: u64,         // 最大延遲（毫秒），預設 5000
    pub backoff_multiplier: f64,   // 退避倍數，預設 2.0
}

impl Default for RetryConfig {
    fn default() -> Self {
        Self {
            max_retries: 3,
            initial_delay_ms: 100,
            max_delay_ms: 5000,
            backoff_multiplier: 2.0,
        }
    }
}
```

pub async fn with_retry<T, F, E>(

## 使用範例

### Retry with Backoff

```rust
use crate::msgq::{RetryConfig, with_retry};

let config = RetryConfig {
    max_retries: 3,
    initial_delay_ms: 100,
    max_delay_ms: 5000,
    backoff_multiplier: 2.0,
};

// 指數退避：100ms -> 200ms -> 400ms -> 800ms
let result = with_retry(config, || {
    flaky_operation()
}).await;

match result {
    Ok(value) => println!("成功: {:?}", value),
    Err(e) => println!("重試失敗: {:?}", e),
}
```

### Exactly-Once

```rust
use crate::msgq::{AsyncQueue, ExactlyOnceQueue};

let mut queue = AsyncQueue::new("orders");
let mut eoq = ExactlyOnceQueue::new(queue, 3600);

// 第一次調用 - 成功入隊
let id1 = eoq.enqueue_once("order-123".to_string(), b"order data".to_vec()).await?;
assert!(id1.is_some());

// 相同 key 第二次調用 - 返回 None（已處理過）
let id2 = eoq.enqueue_once("order-123".to_string(), b"order data".to_vec()).await?;
assert!(id2.is_none());

// 消費訊息
let msg = eoq.dequeue(0).await?.unwrap();
println!("處理訊息: {:?}", msg.payload_str());

// 確認處理
eoq.ack(&msg.id).await?;

// 清理過期的 keys（可定時呼叫）
let removed = eoq.cleanup_expired().await?;
println!("清理了 {} 個過期 keys", removed);
```

## 設計說明

### Idempotency Key 格式

建議使用有意義的 key：
- `order-123` - 訂單 ID
- `payment-456` - 付款 ID
- `user:action:timestamp` - 組合 key

key 會被儲存，設計時需考慮 TTL。

### TTL 設計

```rust
// TTL 建議
let eoq = ExactlyOnceQueue::new(queue, 3600); // 1 小時
let eoq = ExactlyOnceQueue::new(queue, 86400); // 24 小時
```

TTL 應根據業務需求設定：
- 短 TTL（幾分鐘）：高頻、重複少的操作
- 長 TTL（幾小時）：低頻、可能延遲確認的操作
