# v3.15 - 資料庫管理（VACUUM / ANALYZE / BACKUP）

## 日期：2026-05-17

## 目標

在 SQL Executor 模組中，實作三個重要的資料庫管理指令：
- `VACUUM` — 回收未使用的磁碟空間，重整資料庫檔案
- `ANALYZE` — 收集資料表統計資訊，供查詢最佳化器使用
- `BACKUP` — 將資料庫備份至外部檔案

## 指令語法

```sql
VACUUM;
ANALYZE [TABLE table_name];
BACKUP TO 'backup_path';
```

## 已實作

### 1. AST 結構與解析支援

在 parser 的 AST 中新增對應結構：

```rust
VacuumStmt {}

AnalyzeStmt {
    table_name: Option<String>,
}

    path: String,

```

解析器支援以下語法：
- `VACUUM` — 完整資料庫重整
- `ANALYZE` — 全域統計資訊收集
- `ANALYZE table_name` — 指定資料表統計資訊收集
- `BACKUP TO 'path'` — 備份至指定路徑

```rust
pub struct Executor {

    page_count: u64,
    avg_row_size: f64,
    index_info: HashMap<String, IndexStats>,
}
```

### 3. 操作指令實作

#### 3.1 VACUUM

`execute_vacuum` 函式負責執行資料庫重整：

```rust
fn execute_vacuum(&mut self) -> Result<()> {
    self.engine.flush()?;
    match self.engine.engine_type() {
        "memory" => {
            // Memory engine 無需重整，純 KV 結構無碎片
            Ok(())
        }
        "btree" => {
            // BTree engine: 重建所有頁面，釋放未使用頁面
            self.btree_vacuum()?;
            Ok(())
        }
        "lsm" => {
            // LSM engine: 觸發 major compaction
            self.lsm_compaction()?;
            Ok(())
        }
        _ => Err(SqlError::EngineNotSupported),
    }
}
```

**Memory Engine**：因使用 `BTreeMap`，無磁碟碎片問題，VACUUM 為 no-op。

**BTree Engine**：拜訪所有 leaf pages，重建 BTree 結構，釋放已刪除資料佔用的頁面。

**LSM Engine**：觸發一次 major compaction，將多層 SSTable 合併，移除已刪除的資料。

#### 3.2 ANALYZE

`execute_analyze` 函式負責收集統計資訊：

```rust
fn execute_analyze(&mut self, table_name: Option<String>) -> Result<()> {
    match table_name {
        Some(name) => self.analyze_table(&name)?,
        None => self.analyze_all_tables()?,
    }
    Ok(())
}

fn analyze_table(&mut self, table_name: &str) -> Result<()> {
    let table_id = self.get_table_id(table_name)?;
    let scan_result = self.engine.scan(table_id, &[], &[])?;

    let row_count = scan_result.len() as u64;
    let total_size: u64 = scan_result.iter()
        .map(|(_, v)| v.len() as u64)
        .sum();
    let avg_row_size = total_size as f64 / row_count as f64;

    let stats = TableStats {
        row_count,
        page_count: (row_count / 100).max(1),
        avg_row_size,
        index_info: HashMap::new(),
    };

    self.stats.table_stats.insert(table_name.to_string(), stats);
    self.stats.last_analyze.insert(table_name.to_string(), Utc::now());

    Ok(())
}
```

ANALYZE 會：
1. Scan 目標資料表的所有資料列
2. 計算 `row_count`、`avg_row_size` 等統計數值
3. 將結果存入 `ExecutorStats`
4. 記錄最後分析時間

#### 3.3 BACKUP

`execute_backup` 函式負責資料庫備份：

```rust
fn execute_backup(&mut self, path: &str) -> Result<()> {
    let backup_path = Path::new(path);

    match self.engine.engine_type() {
        "memory" => {
            // Memory engine: 將所有 table 的 KV pair 序列化輸出
            self.backup_memory_engine(backup_path)?;
        }
        "btree" => {
            // BTree engine: 複製所有 data file + WAL
            self.backup_btree_engine(backup_path)?;
        }
        "lsm" => {
            // LSM engine: 複製所有 SSTable + WAL
            self.backup_lsm_engine(backup_path)?;
        }
        _ => return Err(SqlError::EngineNotSupported),
    }

    Ok(())
}

fn backup_memory_engine(&self, path: &Path) -> Result<()> {
    let mut file = File::create(path)?;
    let mut backup = MemoryBackup {
        version: "db6.backup.v1".to_string(),
        timestamp: Utc::now(),
        tables: HashMap::new(),
    };

    for (&table_id, table_data) in self.engine.iter_tables() {
        let mut records = Vec::new();
        for (key, value) in table_data.scan(&[], &[])? {
            records.push((key, value));
        }
        backup.tables.insert(table_id, records);
    }

    let json = serde_json::to_vec(&backup).map_err(|e| SqlError::BackupFailed(e.to_string()))?;
    file.write_all(&json)?;

    Ok(())
}
```

BACKUP 會：
1. 根據引擎類型選擇對應備份策略
2. Memory engine：將所有 table 的 KV 序列化為 JSON 格式
3. BTree/LSM engine：複製底層資料檔案（未來實作）
4. 備份檔案包含版本號與時間戳

### 4. REPL 整合

在 REPL 中新增對應指令：

```rust
match cmd.trim() {
    "VACUUM" => executor.execute_vacuum()?,
    "ANALYZE" => executor.execute_analyze(None)?,
    cmd if cmd.starts_with("ANALYZE ") => {
        let table = cmd.trim_start_matches("ANALYZE ");
        executor.execute_analyze(Some(table))?;
    }
    cmd if cmd.starts_with("BACKUP TO ") => {
        let path = cmd.trim_start_matches("BACKUP TO ").trim_matches('\'');
        executor.execute_backup(path)?;
    }
    _ => { ... }
}
```

```sql
-- 3. ANALYZE — 收集特定資料表統計
ANALYZE users;

-- 4. BACKUP — 備份至檔案
BACKUP TO '/tmp/db6_backup.json';
```

## 現有限制與未來優作

1. **BTree/LSM BACKUP 尚未完整實作**
   當前 BACKUP TO 對 BTree 與 LSM 引擎僅輸出框架，實際的檔案複製尚未完成。未來將移植 sql6 的 backup 機制。

2. **ANALYZE 統計資訊尚未用於查詢最佳化**
   目前 `TableStats` 僅做收集，尚未影響查詢規劃（如 join order、index selection）。下一步將整合 Planner 使用統計資訊。

3. **無 INCREMENTAL BACKUP**
   目前僅支援完整備份，未來可加入增量備份（只備份變更部分）以提升大型資料庫的備份效率。

4. **VACUUM 為同步阻塞操作**
   對 LSM 發起 major compaction 可能耗時較長，未來可改為非同步背景執行。
