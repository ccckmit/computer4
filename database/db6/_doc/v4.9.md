# v4.9 SQL Queue 優化 (mini-redis 模式)

## 規劃日期
2026-05-17

## 目標

參考 mini-redis server 架構，優化 SQL Queue 的並發處理。

## 差異分析

| mini-redis | 目前的 SQL Queue | 改進方向 |
|------------|------------------|----------|
| Per-connection task | Sequential workers | 每個 SQL 獨立 task |
| Semaphore 限制連線數 | 無限制 | Semaphore 限制並發數 |
| Graceful shutdown | 無 | broadcast channel 優雅關閉 |
| Backoff 重試策略 | 無 | 失敗時指數退避 |
| select! 監聽多事件 | 簡單 loop | tokio::select! 多任務監聽 |

## 改進內容

### 1. 每個 SQL 獨立 Task（取代 Worker Pool）

```rust
// 改前：Worker pool（固定數量，sequential 處理）
pub fn spawn_workers(&self, count: usize) {
    for _ in 0..count {
        tokio::spawn(async move { worker_loop(...) });
    }
}

// 改後：每個 SQL 立即 spawn 獨立 task（真正並發）
pub async fn execute(&self, sql: &str) -> Result<String, String> {
    let job = SqlJob::new(sql.to_string());
    let job_id = job.job_id.clone();
    
    let results = self.results.clone();
    
    // 每個 SQL 獨立 task
    tokio::spawn(async move {
        let result = execute_sql(&job.sql).await;
        if let Err(e) = results.store(&job.job_id, result).await {
            eprintln!("Failed to store result: {}", e);
        }
    });
    
    Ok(job_id)
}
```

### 2. Semaphore 限制並發數

```rust
pub struct AsyncSqlExecutor {
    queue: AsyncQueue,
    results: ResultStore,
    semaphore: Arc<Semaphore>,  // 新增
    shutdown: broadcast::Sender<()>,  // 新增
}

impl AsyncSqlExecutor {
    pub fn new(concurrency_limit: usize) -> Self {
        Self {
            queue: AsyncQueue::new("sql"),
            results: ResultStore::memory(),
            semaphore: Arc::new(Semaphore::new(concurrency_limit)),
            shutdown: broadcast::channel(1).0,
        }
    }
    
    pub async fn execute(&self, sql: &str) -> Result<String, String> {
        // 取得許可
        let permit = self.semaphore.acquire().await.map_err(|e| e.to_string())?;
        
        let job = SqlJob::new(sql.to_string());
        let job_id = job.job_id.clone();
        
        let results = self.results.clone();
        
        tokio::spawn(async move {
            let result = execute_sql(&job.sql).await;
            let _ = results.store(&job.job_id, result).await;
            drop(permit);  // 釋放名額
        });
        
        Ok(job_id)
    }
}
```

### 3. Graceful Shutdown

```rust
pub struct AsyncSqlExecutor {
    shutdown: broadcast::Sender<()>,
}

impl AsyncSqlExecutor {
    pub fn shutdown(&self) {
        // 廣播關閉訊號
        let _ = self.shutdown.send(());
    }
    
    pub async fn wait_until_shutdown(&self) {
        let mut rx = self.shutdown.subscribe();
        rx.recv().await;
    }
}

// Worker 監聽 shutdown
async fn worker_loop(
    mut queue: AsyncQueue,
    results: ResultStore,
    shutdown: broadcast::Receiver<()>,
) {
    loop {
        tokio::select! {
            msg = queue.dequeue(0) => { ... }
            _ = shutdown.recv() => {
                // 收到關閉訊號，優雅退出
                break;
            }
        }
    }
}
```

### 4. Backoff 重試（可選）

```rust
async fn execute_with_backoff(sql: &str) -> JobResult {
    let mut backoff = 1;
    loop {
        match execute_sql(sql).await {
            Ok(result) => return result,
            Err(e) if backoff < 64 => {
                tokio::time::sleep(Duration::from_secs(backoff)).await;
                backoff *= 2;
            }
            Err(e) => return JobResult::Error { message: e },
        }
    }
}
```

## 使用範例

```rust
// 建立 executor，限制最多 50 個並發 SQL
let executor = AsyncSqlExecutor::with_concurrency_limit(50);

// 提交多個 SQL（每個立即 spawn 獨立 task）
let job_ids = vec![
    executor.execute("SELECT * FROM users").await?,
    executor.execute("INSERT INTO orders VALUES (1)").await?,
    executor.execute("UPDATE products SET stock = stock - 1").await?,
];

// 輪詢結果
for job_id in job_ids {
    loop {
        match executor.poll(&job_id).await {
            Ok(result) => {
                println!("結果: {}", result.to_json());
                break;
            }
            Err(_) => tokio::time::sleep(Duration::from_millis(50)).await,
        }
    }
}

// 優雅關閉
executor.shutdown();
executor.wait_until_shutdown().await;
println!("Server shutdown complete");
```

- **更高並發**：每個 SQL 立即處理，不用等待 worker
- **資源控制**：Semaphore 避免過多並發造成系統過載
- **優雅關閉**：支援 graceful shutdown
- **更 tokio**：符合 tokio 最佳實踐
