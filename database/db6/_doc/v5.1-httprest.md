# v5.1 HTTP/REST Transport

## Overview

Added HTTP/REST transport layer for db6, allowing clients in any language (Python, Node.js, Go, etc.) to access db6 via standard HTTP.

## Features

- HTTP/1.1 + JSON
- Standard REST API design
- Works with curl, fetch, requests, etc.
- Multi-language support

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/health` | Health check |
| POST | `/kv/put` | Put a key-value pair |
| POST | `/kv/get` | Get value by key |
| POST | `/kv/delete` | Delete a key |
| POST | `/kv/scan` | Range scan [start, end) |
| POST | `/kv/batch_put` | Batch write multiple pairs |
| POST | `/kv/range_delete` | Range delete |
| GET | `/kv/stats` | Engine statistics |
| POST | `/sql/execute` | Execute SQL |
| GET | `/ws` | WebSocket upgrade |

## Request/Response Format

### PUT

```bash
curl -X POST http://localhost:50052/kv/put \
  -H "Content-Type: application/json" \
  -d '{"table_id":1,"key":"user:1","value":"{\"name\":\"John\"}"}'
```

Response:
```json
{"status":"ok"}
```

### GET

```bash
curl -X POST http://localhost:50052/kv/get \
  -H "Content-Type: application/json" \
  -d '{"table_id":1,"key":"user:1"}'
```

Response:
```json
{"value":"{\"name\":\"John\"}","found":true}
```

### SCAN

```bash
curl -X POST http://localhost:50052/kv/scan \
  -H "Content-Type: application/json" \
  -d '{"table_id":1,"start":"user:","end":"user:~"}'
```

Response:
```json
{"pairs":[{"key":"user:1","value":"..."},{"key":"user:2","value":"..."}]}
```

### STATS

```bash
curl http://localhost:50052/kv/stats
```

Response:
```json
{"key_count":10,"size_bytes":1024,"engine":"memory-hash"}
```

### SQL

```bash
curl -X POST http://localhost:50052/sql/execute \
  -H "Content-Type: application/json" \
  -d '{"sql":"SELECT * FROM users WHERE age > 18"}'
```

Response:
```json
{"columns":["id","name","age"],"rows":[["1","John","20"]],"affected":0,"error":null}
```

## Usage

```bash
# Start server
cargo run --bin server

# Server listens on http://127.0.0.1:50052
```

## Dependencies

```toml
axum = { version = "0.7", features = ["json"] }
tower = "0.5"
```

## Implementation

- `src/server/http.rs` - HTTP router and handlers
- `src/server/mod.rs` - Module exports
- `src/server_main.rs` - Server entry point

## Notes

- Default port: 50052
- JSON is UTF-8 encoded
- Keys and values are base64 in binary mode, string in text mode
- `table_id` allows multi-table isolation (default: 1)
