# v3.8 - 索引加速

## 日期：2026-05-16

## 目標

在 JSON 過濾的基礎上，新增 secondary index 加速常見欄位的查詢。

## 背景

### 現有問題
```rust
db.table("users").put(b"1", r#"{"name":"Alice","age":30}"#.as_bytes()).unwrap();
// ... 百萬筆資料

// 查詢 age > 25
db.select("*")
    .from("users")
    .filter("$.age > 25")
    .execute();
// 必須遍歷所有資料，全表掃描 O(n)
```

### 索引加速
```rust
// 建立索引
db.create_index("users", "age");

// 查詢優化
db.select("*")
    .from("users")
    .filter("$.age > 25")
    .execute();
// 使用索引，直接找到 age > 25 的 keys，O(log n) 或 O(k)
```

## API 設計

### 建立索引

```rust
// 在 age 欄位上建立索引
db.create_index("users", "age").unwrap();

// 在多個欄位上建立索引
db.create_index("users", "city").unwrap();
db.create_index("users", "name").unwrap();
```

### 刪除索引

```rust
db.drop_index("users", "age").unwrap();
```

### 查詢（自動使用索引）

```rust
// 假設已在 age 上建立索引
let result = db.select("*")
    .from("users")
    .filter("$.age > 25")
    .execute();
// 自動使用 age 索引加速
```

### 列出索引

```rust
// 列出表的索引
let indexes = db.indexes("users").unwrap();
// 返回 ["age", "city", "name"]
```

## 適用範圍

| 引擎 | 支援索引 |
|------|----------|
| BTree (磁碟) | ✅ |
| LSM (磁碟) | ✅ |
| BTreeMemory (`btree-mem`) | ✅ |
| HashMemory (`memory`, `hash`) | ❌ (無序，索引無意義) |

## 實作方式

### 1. 索引資料結構

```rust
// Secondary Index - field_value -> Vec<key>
// 將 JSON 欄位值映射到 keys

pub struct SecondaryIndex {
    table_id: u32,
    path: String,                    // JSON 路徑，如 "$.age"
    index: BTreeMap<String, Vec<Vec<u8>>>,  // value -> keys
}
```

### 2. Db 新增方法

```rust
impl Db {
    pub fn create_index(&mut self, table: &str, json_path: &str) -> Result<()>;
    pub fn drop_index(&mut self, table: &str, json_path: &str) -> Result<()>;
    pub fn indexes(&self, table: &str) -> Result<Vec<String>>;
}
```

### 3. 查詢時自動選擇索引

```rust
// 在 filter_rows 中
fn filter_rows(rows, condition) {
    if condition.starts_with("$.") {
        // 嘗試使用索引
        if let Some(index) = find_index(table_id, &path) {
            // 使用索引獲取候選 keys
            // 只讀取這些 keys 的資料
        }
        // 如果沒有索引，全表掃描
    }
}
```

### 4. 索引維護

```rust
// 插入時自動更新索引
db.table("users").put(b"key", json_bytes).unwrap();
// 自動解析 json_bytes，提取索引欄位值，更新索引

// 刪除時自動移除索引
db.table("users").delete(b"key").unwrap();
// 自動從索引中移除 key
```

## 不做的事項

- ❌ 複合索引 (multi-field index)
- ❌ 唯一索引 (unique index)
- ❌ 索引統計資訊維護 (ANALYZE)

只做簡單的單欄位索引。

## 測試

```rust
#[test]
fn test_index_basic() {
    let mut db = Db::new("btree").unwrap();

    // 建立索引
    db.create_index("users", "age").unwrap();

    // 插入資料
    db.table("users").put(b"1", r#"{"name":"Alice","age":30}"#.as_bytes()).unwrap();
    db.table("users").put(b"2", r#"{"name":"Bob","age":25}"#.as_bytes()).unwrap();
    db.table("users").put(b"3", r#"{"name":"Charlie","age":35}"#.as_bytes()).unwrap();

    // 查詢應該更快
    let result = db.select("*")
        .from("users")
        .filter("$.age > 27")
        .execute()
        .unwrap();

    assert_eq!(result.rows.len(), 2);
}

#[test]
fn test_index_on_drop() {
    let mut db = Db::new("btree").unwrap();

    db.create_index("users", "age").unwrap();
    db.table("users").put(b"1", r#"{"name":"Alice","age":30}"#.as_bytes()).unwrap();

    // 刪除記錄時，索引自動更新
    db.table("users").delete(b"1").unwrap();

    // 驗證索引已移除
    let result = db.select("*")
        .from("users")
        .filter("$.age > 25")
        .execute()
        .unwrap();

    assert_eq!(result.rows.len(), 0);
}
```
