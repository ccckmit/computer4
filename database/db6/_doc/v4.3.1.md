## 已實作項目

### 1. serde_bytes 支援

**改進**: 使用 `serde_bytes` 優化 `Vec<u8>` 序列化效率

```rust
use serde_bytes;

#[derive(Serialize, Deserialize)]
pub struct Message {
    #[serde(with = "serde_bytes")]
    pub payload: Vec<u8>,
}
```

**影響檔案**:
- `src/msgq/message.rs` - SyncQueueMessage
- `src/msgq/async_queue.rs` - AsyncQueueMessage
- `src/msgq/async_pubsub.rs` - AsyncPubSubMessage

### 2. Broadcast Channel 容量參數化

**改進**: PubSubConfig 加入 channel_capacity

```rust
pub struct PubSubConfig {
    pub max_history: usize,
    pub history_enabled: bool,
    pub pattern_matching: bool,
    pub channel_capacity: usize,  // 新增，可設定 1024
}

impl Default for PubSubConfig {
    fn default() -> Self {
        Self {
            max_history: 100,
            history_enabled: true,
            pattern_matching: true,
            channel_capacity: 1024,
        }
    }
}
```

**影響檔案**:
- `src/msgq/sync_pubsub.rs`
- `src/msgq/async_pubsub.rs`

## 未實作項目（複雜度較高）

| 項目 | 說明 | 原因 |
|------|------|------|
| Bytes 類型 | 改用 bytes::Bytes 取代 Vec<u8> | 與 serde_json JSON 序列化衝突，需要 custom serialization |
| Background Purge Task | BTreeSet + Notify 追蹤過期訊息 | 需大幅重構 AsyncQueue 架構 |
| std::sync::Mutex | 取代 tokio::sync::RwLock | async 方法需要 tokio RwLock，std Mutex 會阻塞 |

## 不變更項目

- **不動 kv/ 模組**：這些優化都是 msgq/ 內部實作
- **維持 SyncQueue 使用 KV**：SyncQueue 繼續使用 KV 儲存
