# v3.2 完成項目（FTS 整合）

## 新功能

### KvEngine 實作 StorageEngine Trait

將 `KvEngine` 從單純的 `KvStore` 實作擴展為同時實作 `StorageEngine` trait，使 `FtsIndex<E: StorageEngine>` 可以直接使用 `KvEngine`。

#### 架構變更

- 使用 `Arc<RwLock<T>>` 包裝各類 engine，解決 interior mutability 問題
- `StorageEngine` 和 `KvStore` 兩個 trait 的方法簽名相似，測試中需使用 `KvStore::put()` 明確呼叫

```rust
// 之前
pub enum KvEngine {
    Hash(HashMemoryEngine),
    BTreeMem(BTreeMemoryEngine),
    BTree(BTreeEngine),
    Lsm(LsmEngine),
}

// 現在
pub enum KvEngine {
    Hash(Arc<RwLock<HashMemoryEngine>>),
    BTreeMem(Arc<RwLock<BTreeMemoryEngine>>),
    BTree(Arc<RwLock<BTreeEngine>>),
    Lsm(Arc<RwLock<LsmEngine>>),
}
```

### FtsIndex<KvEngine> 可用

現在可以直接使用 `FtsIndex<KvEngine>`：

```rust
use db6::{KvEngine, FtsIndex, StorageEngine};

let engine = KvEngine::new("memory").unwrap();
let mut index = FtsIndex::new(engine);

index.insert(1, "Hello World").unwrap();
index.insert(2, "資料庫系統").unwrap();

let results = index.search("Hello").unwrap();  // returns [1]
let results2 = index.search("資料庫").unwrap(); // returns [2]
```

支援的 search 方法：
- `search(query)` - 基本搜尋
- `search_prefix(prefix)` - 前綴搜尋
- `search_boolean(query)` - 布林搜尋（AND/OR/NOT）
- `search_bm25(query)` - BM25 排序
