# db6 Communication Architecture

## Overview

db6 provides multiple transport layer options for client access:

```
                    ┌─────────────────────────────────────────────────┐
                    │              db6 Server                         │
                    │  ┌─────────────────────────────────────────┐  │
                    │  │           Transport Layer               │  │
                    │  ├─────────┬─────────┬─────────┬────────────┤  │
                    │  │  gRPC   │   HTTP  │WebSocket│    UDS     │  │
                    │  │ (port   │  (port  │ (port   │  /tmp/db6  │  │
                    │  │  50051) │   50052)│  50053) │   .sock    │  │
                    │  └─────────┴─────────┴─────────┴────────────┘  │
                    │  ┌─────────────────────────────────────────┐  │
                    │  │           Unified Handler                │  │
                    │  │  (KV + SQL + FTS + Msgq unified API)     │  │
                    │  └─────────────────────────────────────────┘  │
                    └─────────────────────────────────────────────────┘
```

## Architecture

```
src/
├── lib.rs                    # Core library exports
├── main.rs                   # REPL entry point
├── server/                   # New: Multi-transport server
│   ├── mod.rs
│   ├── server.rs             # Main server (spawns all transports)
│   ├── grpc.rs               # gRPC transport (tonic)
│   ├── http.rs               # HTTP/REST transport (axum)
│   ├── websocket.rs           # WebSocket transport (tungstenite)
│   └── uds.rs                # UNIX Domain Socket transport
└── proto/                    # Protocol definitions
    └── db6.proto             # gRPC service definitions
```

## Transport Options

| Transport | Port/Path | Protocol | Use Case | Client Support |
|-----------|-----------|----------|----------|----------------|
| **HTTP** | 50052 | HTTP/1.1 + JSON | Simple REST, browser, curl | Any HTTP client |
| **WebSocket** | 50052 | WebSocket + JSON | Real-time, streaming | ws, WebSocket API |
| **gRPC** | 50051 | HTTP/2 + Protobuf | High-performance RPC, multi-language | grpc-java, grpcio, go-grpc |
| **UDS** | /tmp/db6.sock | Unix Socket | Local processes, high performance | Native Unix |

## Protocol Design

### Common Message Format

All transports expose the same API:

```
KV Operations:
- put(table_id, key, value) -> ()
- get(table_id, key) -> Option<value>
- delete(table_id, key) -> ()
- scan(table_id, start, end) -> Vec<(key, value)>
- batch_put(table_id, Vec<(key, value)>) -> ()
- range_delete(table_id, start, end) -> ()

SQL Operations:
- execute(sql) -> ResultSet

FTS Operations:
- insert(doc_id, text) -> ()
- search(query) -> Vec<doc_id>
- search_bm25(query) -> Vec<(doc_id, score)>

Msgq Operations:
- queue_create(name) -> ()
- enqueue(queue, payload, timeout) -> msg_id
- dequeue(queue, wait_timeout) -> Option<Message>
- ack(queue, msg_id) -> ()
- pubsub_publish(channel, payload) -> msg_id
- pubsub_subscribe(channel) -> Stream<Message>
```

## gRPC (Port 50051)

### Proto Definition

```protobuf
syntax = "proto3";

package db6;

service Db6Service {
  // KV Operations
  rpc Put(PutRequest) returns (PutResponse);
  rpc Get(GetRequest) returns (GetResponse);
  rpc Delete(DeleteRequest) returns (DeleteResponse);
  rpc Scan(ScanRequest) returns (ScanResponse);

  // SQL Operations
  rpc ExecuteSql(SqlRequest) returns (SqlResponse);

  // FTS Operations
  rpc InsertDoc(FtsInsertRequest) returns (FtsInsertResponse);
  rpc Search(SearchRequest) returns (SearchResponse);
  rpc SearchBm25(SearchRequest) returns (SearchBm25Response);
}

message PutRequest { uint32 table_id = 1; bytes key = 2; bytes value = 3; }
message PutResponse {}
message GetRequest { uint32 table_id = 1; bytes key = 2; }
message GetResponse { bytes value = 1; bool found = 2; }
```

### Client Example (Python)

```python
import grpc
import db6_pb2, db6_pb2_grpc

channel = grpc.insecure_channel('localhost:50051')
stub = db6_pb2_grpc.Db6ServiceStub(channel)

# KV Put
req = db6_pb2.PutRequest(table_id=1, key=b'key', value=b'value')
stub.Put(req)

# KV Get
req = db6_pb2.GetRequest(table_id=1, key=b'key')
resp = stub.Get(req)
print(resp.value, resp.found)
```

## HTTP/REST (Port 50052)

### Endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | /kv/put | Put a key-value pair |
| POST | /kv/get | Get value by key |
| POST | /kv/delete | Delete a key |
| POST | /kv/scan | Range scan |
| POST | /kv/batch_put | Batch write |
| GET | /kv/stats | Engine statistics |
| POST | /sql/execute | Execute SQL |
| POST | /fts/insert | Insert document |
| POST | /fts/search | Full-text search |

### Request/Response Format

```json
// POST /kv/put
{
  "table_id": 1,
  "key": "user:123",
  "value": "{\"name\": \"John\"}"
}

// Response
{ "status": "ok" }

// GET /kv/get
{ "table_id": 1, "key": "user:123" }
// Response
{ "value": "{\"name\": \"John\"}", "found": true }
```

### Client Example (any language)

```bash
curl -X POST http://localhost:50052/kv/put \
  -H "Content-Type: application/json" \
  -d '{"table_id":1,"key":"test","value":"hello"}'
```

## WebSocket (Port 50053)

### Protocol

WebSocket provides bidirectional, real-time communication:

```javascript
// Client (JavaScript)
const ws = new WebSocket('ws://localhost:50053');

// Send request
ws.send(JSON.stringify({
  method: 'kv_put',
  params: { table_id: 1, key: 'test', value: 'hello' }
}));

// Receive response
ws.onmessage = (event) => {
  const resp = JSON.parse(event.data);
  console.log(resp);
};

// Subscribe to real-time updates
ws.send(JSON.stringify({
  method: 'subscribe',
  params: { channel: 'updates' }
}));
```

### Streaming Support

WebSocket is ideal for:
- Real-time database updates
- Pub/Sub notifications
- Long-running query results

## UNIX Domain Socket (/tmp/db6.sock)

### Purpose

- Highest performance for local processes
- No network overhead
- Secure (only local access)

### Client Example (Rust)

```rust
use std::os::unix::net::UnixStream;
use std::io::{Read, Write};

let mut stream = UnixStream::connect("/tmp/db6.sock").unwrap();

// Send request
let req = serde_json::to_vec(&request).unwrap();
stream.write_all(&req).unwrap();

// Read response
let mut buf = Vec::new();
stream.read_to_end(&mut buf).unwrap();
let resp: Response = serde_json::from_slice(&buf).unwrap();
```

## Configuration

Environment variables:
- `DB6_SERVER_HOST` - Bind address (default: 127.0.0.1)
- `DB6_GRPC_PORT` - gRPC port (default: 50051)
- `DB6_HTTP_PORT` - HTTP port (default: 50052)
- `DB6_WS_PORT` - WebSocket port (default: 50053)
- `DB6_UDS_PATH` - UNIX socket path (default: /tmp/db6.sock)

## Performance Characteristics

| Transport | Latency | Throughput | Use Case |
|-----------|---------|-------------|----------|
| UDS | ~0.01ms | Highest | Local, latency-critical |
| gRPC | ~0.1ms | High | Remote, multi-language |
| HTTP | ~0.2ms | Medium | Web, simple clients |
| WebSocket | ~0.2ms | Medium | Real-time, streaming |

## Security Considerations

- UDS: Only local access, no authentication needed
- gRPC/HTTP/WebSocket: TLS recommended for production
- All: Input validation on all requests
- All: Rate limiting to prevent abuse
