# v4.1 - Pub/Sub 發布/訂閱模式

## 日期：2026-05-17

## 目標

在現有的 msgq 模組中新增 Pub/Sub 功能，支援一訊息多訂閱者（broadcast）。

## Sync vs Async 明確區分

| 類型 | 實作 | 檔案 |
|------|------|------|
| **Sync** | `SyncQueue` + `SyncPubSub` | 使用 KV 儲存 |
| **Async** | `AsyncQueue` + `AsyncPubSub` | 使用 tokio channel |

## 差異說明

| 特性 | Queue（佇列） | Pub/Sub（發布/訂閱） |
|------|--------------|---------------------|
| 訊息歸屬 | 一個 consumer 取走後其他人看不到 | 所有訂閱者都能收到 |
| 處理模型 | FIFO，一次取走 | Broadcast，複製給所有人 |
| 確認機制 | ack/nack | 無需 ack（廣播模式） |
| 適用場景 | 工作排程 | 通知、事件廣播 |

## Key 設計（SyncPubSub）

```
pubsub:{channel}:messages    -> 訊息列表（append-only）
pubsub:{channel}:subscribers -> 訂閱者列表
pubsub:{channel}:offset:{subscriber_id} -> 訂閱者偏移量
```

### SyncPubSubMessage 結構

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SyncPubSubMessage {
    pub id: String,
    pub channel: String,
    pub payload: Vec<u8>,
    pub timestamp: u64,
}
```

## Sync API 設計

```rust
pub struct SyncPubSub {
    name: String,
    engine: Arc<RwLock<KvEngine>>,
}

impl SyncPubSub {
    /// 建立 SyncPubSub 實例
    pub fn new(name: &str, engine: Arc<RwLock<KvEngine>>) -> Self;

    /// 發布訊息到頻道
    pub fn publish(&mut self, channel: &str, payload: Vec<u8>) -> Result<String>;

    /// 訂閱頻道
    pub fn subscribe(&mut self, channel: &str, subscriber_id: &str) -> Result<()>;

    /// 取消訂閱
    pub fn unsubscribe(&mut self, channel: &str, subscriber_id: &str) -> Result<()>;

    /// 消費訊息（取得新訊息）
    pub fn consume(&mut self, channel: &str, subscriber_id: &str) -> Result<Option<SyncPubSubMessage>>;

    /// 列出所有頻道
    pub fn list_channels(&self) -> Result<Vec<String>>;

    /// 列出頻道的訂閱者
    pub fn list_subscribers(&self, channel: &str) -> Result<Vec<String>>;

    /// 取得頻道訊息數
    pub fn message_count(&self, channel: &str) -> Result<usize>;
}
```

## Async API 設計

```rust
pub struct AsyncPubSub {
    channels: Arc<RwLock<HashMap<String, broadcast::Sender<AsyncPubSubMessage>>>>,
}

impl AsyncPubSub {
    /// 發布訊息到頻道
    pub async fn publish(&self, channel: &str, payload: Vec<u8>) -> Result<String, String>;

    /// 訂閱頻道 - 返回 receiver
    pub async fn subscribe(&self, channel: &str) -> Result<broadcast::Receiver<AsyncPubSubMessage>, String>;

    /// 列出所有頻道
    pub async fn list_channels(&self) -> Vec<String>;

    /// 取得訂閱者數
    pub async fn subscriber_count(&self, channel: &str) -> usize;
}
```

## 使用範例

### Sync 版本

```rust
use db6::{Msgq, SyncPubSub, SyncPubSubMessage};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let msgq = Msgq::new("memory")?;
    let mut ps = msgq.pubsub();

    // 訂閱者 A 訂閱 "notifications" 頻道
    ps.subscribe("notifications", "subscriber_A")?;

    // 訂閱者 B 訂閱 "notifications" 頻道
    ps.subscribe("notifications", "subscriber_B")?;

    // 發布訊息
    let msg_id = ps.publish("notifications", b"Hello all subscribers!".to_vec())?;
    println!("Published: {}", msg_id);

    // 訂閱者 A 消費
    let msg_a = ps.consume("notifications", "subscriber_A")?;
    println!("A received: {:?}", msg_a.payload_str());

    // 訂閱者 B 消費（同一則訊息）
    let msg_b = ps.consume("notifications", "subscriber_B")?;
    println!("B received: {:?}", msg_b.payload_str());

    Ok(())
}
```

### Async 版本

```rust
use db6::{AsyncPubSub, AsyncPubSubMessage};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let ps = AsyncPubSub::new();

    // 訂閱得到 receiver
    let mut sub1 = ps.subscribe("news").await.unwrap();
    let mut sub2 = ps.subscribe("news").await.unwrap();

    // 發布
    let id = ps.publish("news", b"Hello".to_vec()).await.unwrap();

    // 兩訂閱者都能收到（broadcast）
    let msg1 = sub1.recv().await.unwrap();
    let msg2 = sub2.recv().await.unwrap();

    println!("1: {:?}, 2: {:?}", msg1.payload_str(), msg2.payload_str());

    Ok(())
}
```

## 實作細節（SyncPubSub）

### 1. 訊息儲存

```rust
#[derive(Serialize, Deserialize)]
struct ChannelMessages {
    messages: Vec<SyncPubSubMessage>,
}
```

- 新訊息 push 到 `messages` 陣列末端
- 每個訂閱者有獨立的 offset

### 2. 訂閱者offset追蹤

```rust
/// 訂閱者偏移量
struct SubscriberOffset {
    subscriber_id: String,
    offset: usize,  // 已讀到的訊息位置
}
```

### 3. consume 邏輯

```rust
fn consume(&mut self, channel: &str, subscriber_id: &str) -> Result<Option<SyncPubSubMessage>> {
    let offset = self.get_offset(channel, subscriber_id)?;
    let messages = self.get_channel_messages(channel)?;

    if offset < messages.len() {
        let msg = messages[offset].clone();
        self.set_offset(channel, subscriber_id, offset + 1)?;
        return Ok(Some(msg));
    }

    Ok(None)
}
```

## 測試案例（Sync）

```rust
#[test]
fn test_sync_pubsub_basic() {
    let msgq = Msgq::new("memory").unwrap();
    let mut ps = msgq.pubsub();

    // 訂閱
    ps.subscribe("news", "reader1").unwrap();
    ps.subscribe("news", "reader2").unwrap();

    // 發布
    let id = ps.publish("news", b"Breaking news!".to_vec()).unwrap();
    assert!(!id.is_empty());

    // 兩訂閱者都能收到
    let msg1 = ps.consume("news", "reader1").unwrap().unwrap();
    let msg2 = ps.consume("news", "reader2").unwrap().unwrap();

    assert_eq!(msg1.payload, b"Breaking news!");
    assert_eq!(msg2.payload, b"Breaking news!");
    assert_eq!(msg1.id, msg2.id);
}

#[test]
fn test_async_pubsub_basic() {
    let ps = AsyncPubSub::new();

    // 訂閱
    let mut sub1 = ps.subscribe("news").await.unwrap();
    let mut sub2 = ps.subscribe("news").await.unwrap();

    // 發布
    let id = ps.publish("news", b"Hello".to_vec()).await.unwrap();

    // 兩訂閱者都能收到（broadcast）
    let msg1 = sub1.recv().await.unwrap();
    let msg2 = sub2.recv().await.unwrap();

    assert_eq!(msg1.payload, b"Hello");
    assert_eq!(msg2.payload, b"Hello");
}
```

## 與 Msgq 的整合

```rust
pub struct Msgq {
    engine: Arc<RwLock<KvEngine>>,
}

impl Msgq {
    // Sync
    pub fn queue(&self, name: &str) -> SyncQueue { ... }
    pub fn pubsub(&self) -> SyncPubSub { ... }

    // Async
    pub fn async_queue(&self, name: &str) -> AsyncQueue { ... }
    // AsyncPubSub 是獨立的，不需要 engine
}
```
