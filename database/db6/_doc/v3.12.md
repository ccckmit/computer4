# v3.12 - SQL Executor 完整實作

## 日期：2026-05-17

## 目標

完成 SQL Executor 的四個核心操作：INSERT、SELECT、UPDATE、DELETE，針對 BTree 引擎提供完整支援。

## 背景

### 已實作的功能

| 功能 | 狀態 | 說明 |
|------|------|------|
| INSERT | ✅ 完成 | 批量插入，支援 1-2 個值 |
| SELECT | ✅ 完成 | 全表掃描、WHERE 過濾、JOIN、GROUP BY、ORDER BY、LIMIT |
| UPDATE | ✅ 完成 | 支援 WHERE 條件過濾（但 `key` 為保留關鍵字） |
| DELETE | ✅ 完成 | 支援 WHERE 條件過濾（但 `key` 為保留關鍵字） |

### 限制

- 所有表共享 `table_id = 1`，使用 table prefix 模擬表隔離
- `key` 是保留關鍵字，無法作為欄位名在 WHERE 中使用
- 目前只能過濾 JSON 欄位（如 `@.name = 'Alice'`）

## 實作細節

### Table Prefix 掃描

為了解決多表資料隔離的問題，Executor 使用 table prefix 來限定掃描範圍：

```rust
fn get_table_prefix(table_name: &str) -> Vec<u8> {
    format!("{}:", table_name).into_bytes()
}

let table_prefix = get_table_prefix(&table_name);
let table_end = format!("{};", table_name);
let rows = self.engine.scan(1, &table_prefix, table_end.as_bytes())?;
```

### WHERE 條件過濾

UPDATE 和 DELETE 支援 WHERE 條件過濾：

```rust
for (key, val) in rows {
    if let Some(ref where_expr) = delete.where_ {
        if !eval_expr(where_expr, &val) {
            continue;
        }
    }
    self.engine.delete(1, &key)?;
    affected += 1;
}
```

### 限制：`key` 保留關鍵字

SQL Engine 中 `key` 是 `Token::Key` 保留關鍵字，無法直接在 WHERE 中使用：

```sql
-- 無法運作（key 是保留關鍵字）
SELECT * FROM users WHERE key = 'users:1';

-- 可以運作（使用 @.field JSON path）
SELECT * FROM users WHERE @.name = 'Alice';
```

## Executor 結構

### Executor (使用 Box<dyn StorageEngine>)

```rust
pub struct Executor {
    engine: Box<dyn StorageEngine>,
}

impl Executor {
    pub fn new(engine: Box<dyn StorageEngine>) -> Self
    pub fn execute(&mut self, sql: &str) -> Result<ResultSet>
    fn execute_select(&mut self, select: &SelectStmt) -> Result<ResultSet>
    fn execute_insert(&mut self, insert: &InsertStmt) -> Result<ResultSet>
    fn execute_update(&mut self, update: &UpdateStmt) -> Result<ResultSet>
    fn execute_delete(&mut self, delete: &DeleteStmt) -> Result<ResultSet>
}
```

### SqlExecutor<E> (使用泛型引擎)

```rust
pub struct SqlExecutor<E: crate::engine::StorageEngine> {
    engine: E,
}

impl<E: crate::engine::StorageEngine> SqlExecutor<E> {
    pub fn new(engine: E) -> Self
    pub fn execute(&mut self, sql: &str) -> Result<ResultSet>
    pub fn execute_order_by(&mut self, sql: &str) -> Result<ResultSet>
        where E: crate::engine::CanOrderBy
    pub fn execute_fts(&mut self, sql: &str) -> Result<ResultSet>
        where E: crate::engine::CanFts
    pub fn execute_with_tx(&mut self, sql: &str) -> Result<ResultSet>
        where E: crate::engine::CanTransaction
}
```

## 測試案例

### INSERT 測試

```rust
#[test]
fn test_executor_insert_and_select() {
    let engine = crate::engine::BTreeMemoryEngine::new();
    let mut exec = Executor::new(Box::new(engine));

    exec.execute("INSERT INTO test VALUES ('{\"name\":\"Alice\",\"age\":30}')").unwrap();
    let result = exec.execute("SELECT * FROM test").unwrap();

    assert_eq!(result.rows.len(), 1);
    assert_eq!(result.rows[0][1], "{\"name\":\"Alice\",\"age\":30}");
}
```

### UPDATE 測試

```rust
#[test]
fn test_executor_update_with_where() {
    let mut exec = Executor::new(Box::new(engine));

    exec.execute("INSERT INTO t1 VALUES ('1', '{\"name\":\"Alice\"}')").unwrap();
    exec.execute("INSERT INTO t1 VALUES ('2', '{\"name\":\"Bob\"}')").unwrap();

    // UPDATE t1 SET value = 'updated' WHERE @.name = 'Alice'
    let result = exec.execute("UPDATE t1 SET value = 'updated' WHERE @.name = 'Alice'").unwrap();
    assert_eq!(result.affected, 1);

    let select = exec.execute("SELECT * FROM t1").unwrap();
    // Bob should be unchanged, Alice should be updated
}
```

### DELETE 測試

```rust
#[test]
fn test_executor_delete_with_where() {
    let mut exec = Executor::new(Box::new(engine));

    exec.execute("INSERT INTO t1 VALUES ('1', '{\"name\":\"Alice\"}')").unwrap();
    exec.execute("INSERT INTO t1 VALUES ('2', '{\"name\":\"Bob\"}')").unwrap();

    // DELETE FROM t1 WHERE @.name = 'Alice'
    let result = exec.execute("DELETE FROM t1 WHERE @.name = 'Alice'").unwrap();
    assert_eq!(result.affected, 1);

    let select = exec.execute("SELECT * FROM t1").unwrap();
    assert_eq!(select.rows.len(), 1);
}
```
