# v3.13 - 子查詢 (Subquery) 支援

## 日期：2026-05-17

## 目標

在 SQL Executor 中支援子查詢，包括 IN 和 EXISTS 語法。

## 已實作

### 1. 解析器支援（已存在）

```rust
// src/sql/parser/ast.rs

/// IN (SELECT ...) 子查詢
InSubquery {
    expr: Box<Expr>,           // 左邊的表達式，如 @.name
    query: Box<SelectStmt>,    // 子查詢
    negated: bool,             // 是否為 NOT IN
}

/// EXISTS (SELECT ...) 子查詢
Exists {
    query: Box<SelectStmt>,
    negated: bool,
}
```

### 2. 評估函數

```rust
fn eval_expr_with_subqueries(
    expr: &Expr,
    value: &[u8],
    engine: &dyn StorageEngine,
) -> bool {
    match expr {
        Expr::Exists { query, negated } => {
            let values = collect_subquery_values(query, engine);
            let exists = !values.is_empty();
            if *negated { !exists } else { exists }
        }
        Expr::InSubquery { expr, query, negated } => {
            let values = collect_subquery_values(query, engine);
            let left_val = extract_value(expr, value);
            let found = values.contains(&left_val);
            if *negated { !found } else { found }
        }
        _ => eval_expr(expr, value),
    }
}
```

## 測試案例

### EXISTS 子查詢

```sql
-- 簡單 EXISTS（總是返回所有行，因為子查詢不為空）
SELECT * FROM users WHERE EXISTS (SELECT 1);

-- 相關 EXISTS（需根據外層行的值過濾）
SELECT * FROM users u WHERE EXISTS (SELECT 1 FROM orders o WHERE o.user_id = u.id);
```

### IN 子查詢

```sql
-- IN 子查詢
SELECT * FROM users WHERE name IN (SELECT name FROM approved);

-- NOT IN 子查詢
SELECT * FROM users WHERE name NOT IN (SELECT name FROM blocked);
```

## 已知問題

### 1. 欄位提取不正確

`SELECT username FROM table` 目前返回完整的 key-value 對，而非僅提取的欄位值。

原因：`collect_subquery_values` 中的欄位提取邏輯有 bug。

### 2. 相關子查詢不完全支援

對於 `EXISTS (SELECT 1 FROM t2 WHERE t2.id = t1.id)` 這類相關子查詢，目前的實作無法正確處理關聯。

## 限制

- 只支援 BTree 引擎
- 子查詢的欄位提取需要修復
- 相關子查詢（correlated subquery）需要更多工作

## 下一步

- 修復欄位提取邏輯
- 支援相關子查詢
- 實作 `IN (SELECT ...)` 的正確評估
- 實作 `ANY` / `SOME` / `ALL` 子查詢
