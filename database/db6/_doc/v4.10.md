# v4.10 重構規劃

## 目標

系統性重構 msgq 模組，提升架構品質：

1. **拆分大型檔案** - 將 `async_queue.rs` (750+ 行) 拆成多個子模組
2. **統一 Error 處理** - 用 `thiserror` 建立統一錯誤類型
3. **建立 Trait 抽象** - 定義 `Queue` / `PubSub` trait 讓実装可交換

## 完成的項目

### 1. 共用模組 (msgq/common/)

建立 `ConcurrencyLimiter` + `GracefulShutdown` 供所有 async 模組共用：

```rust
// src/msgq/common/mod.rs
pub mod limiter;
pub mod shutdown;

pub use limiter::ConcurrencyLimiter;
pub use shutdown::GracefulShutdown;
pub const DEFAULT_CONCURRENCY_LIMIT: usize = 100;
```

**已套用的模組**：
- `AsyncSqlExecutor` - 使用 ConcurrencyLimiter + GracefulShutdown
- `AsyncQueue` - 新增 `with_config_and_limit()` 建構函式
- `AsyncPubSub` - 新增 `with_config_and_limit()` 建構函式

### 3. Trait 抽象規劃

```rust
// src/msgq/traits.rs
pub trait AsyncMessageQueue: Send + Sync {
    fn enqueue(&self, payload: Vec<u8>, visibility_timeout: u64) -> impl Future<Output = Result<String, String>> + Send;
    fn dequeue(&self, wait_secs: u64) -> impl Future<Output = Result<Option<AsyncQueueMessage>, String>> + Send;
    fn ack(&self, msg_id: &str) -> impl Future<Output = Result<(), String>> + Send;
    fn nack(&self, msg_id: &str) -> impl Future<Output = Result<(), String>> + Send;
    fn length(&self) -> impl Future<Output = Result<usize, String>> + Send;
    fn metrics(&self) -> Arc<QueueMetrics>;
    fn health(&self) -> impl Future<Output = QueueHealth> + Send;
}

pub trait AsyncPubSubTrait: Send + Sync {
    fn publish(&self, channel: &str, payload: Vec<u8>) -> impl Future<Output = Result<String, String>> + Send;
    fn subscribe(&self, channel: &str) -> impl Future<Output = Result<broadcast::Receiver<AsyncPubSubMessage>, String>> + Send;
    fn unsubscribe(&self, channel: &str) -> impl Future<Output = Result<(), String>> + Send;
    fn list_channels(&self) -> impl Future<Output = Vec<String>> + Send;
}
```

### 4. 統一 Error 類型

```rust
// src/msgq/error.rs
use thiserror::Error;

#[derive(Error, Debug)]
pub enum MsgqError {
    #[error("Queue not found: {0}")]
    QueueNotFound(String),

    #[error("Message not found: {0}")]
    MessageNotFound(String),

    #[error("Invalid operation: {0}")]
    InvalidOperation(String),

    #[error("Configuration error: {0}")]
    ConfigError(String),

    #[error("Timeout: {0}")]
    Timeout(String),

    #[error("Channel error: {0}")]
    ChannelError(String),

    #[error("Database error: {0}")]
    Db(String),

    #[error("Invalid engine: {0}")]
    InvalidEngine(String),

    #[error("IO error: {0}")]
    IoError(#[from] std::io::Error),

    #[error("Serialization error: {0}")]
    SerializationError(#[from] serde_json::Error),
}

pub type Result<T> = std::result::Result<T, MsgqError>;
```

## 失敗的項目

重構過程中遇到 error handling 不相容的問題：

1. **sync_queue.rs** 使用 `.map_err(MsgqError::Db)` 但參數類型不符
   - 現有程式碼使用 `fn(String)` 但新錯誤類型需要 `fn(error::Error)`
   
2. **sync_pubsub.rs** 同樣的問題

3. 需要大量修改現有錯誤處理程式碼，決定暫時中止
