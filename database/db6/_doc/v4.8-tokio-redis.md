# v4.8 SQL Queue

## 規劃日期
2026-05-17

## 技術基礎

**建構在 tokio 之上**，使用 msgq/ 現有的 AsyncQueue 和 SyncQueue 實作。

```
msgq/                      (基於 tokio)
├── queue/                 (現有訊息佇列)
│   ├── async_queue.rs    (AsyncQueue - tokio async)
│   └── sync_queue.rs     (SyncQueue - blocking)
├── pubsub/               (現有 Pub/Sub)
└── sql/                  (新增：SQL Queue)
    ├── async_sql.rs      (AsyncSqlExecutor - tokio async)
    └── sync_sql.rs       (SyncSqlExecutor - blocking)
```

**為什麼用 tokio？**
- 原生非同步支援
- 內建 Runtime、Task、Timer
- 與現有 AsyncQueue/AsyncPubSub 一致

## 目標

將 SQL 執行包裝為 Message Queue 服務，類似 PostgreSQL/MySQL 作為網路資料庫服務的運作方式。

## 動機

```
Client → [SQL Queue] → Worker → Database
                    ↑
               Message Queue
               (基於 tokio)
```

這種架構的價值：
- **解耦**：Client 不需要等待 DB 處理完成
- **可靠性**：SQL 不會因連接斷開而丟失
- **重試**：失敗的 SQL 可自動重試
- **負載平衡**：多個 worker 處理請求
- **隔離**：慢查詢不會阻塞其他請求

## 功能清單

### Result Storage 可配置

| 選項 | 說明 | 用途 |
|------|------|------|
| Memory | 記憶體 HashMap | 測試、短期任務 |
| Kv | 使用現有 KV 引擎 | 持久化 |
| Custom | 實作 trait | 使用 Redis/MySQL 等 |

```rust
pub enum ResultStore {
    Memory(Arc<RwLock<HashMap<String, JobResult>>>),
    Kv { engine: KvEngine, prefix: String },
    Custom(Box<dyn SqlResultStore>),
}

pub trait SqlResultStore: Send + Sync {
    fn store(&self, job_id: &str, result: JobResult) -> Result<(), String>;
    fn get(&self, job_id: &str) -> Result<Option<JobResult>, String>;
}
```

### Result 格式

```rust
pub enum JobResult {
    Select {
        columns: Vec<String>,
        rows: Vec<Vec<serde_json::Value>>,
    },
    Insert { affected: u64 },
    Update { affected: u64 },
    Delete { affected: u64 },
    Error { message: String },
}

impl JobResult {
    pub fn to_json(&self) -> serde_json::Value;
}
```


### Async/Sync 雙模式

#### Async 模式 (基於 AsyncQueue + tokio)

```rust
// AsyncSqlExecutor - 非阻塞 (基於 tokio async)
// 提交後立即返回 job_id，結果透過 poll 取得
pub struct AsyncSqlExecutor {
    queue: AsyncQueue,       // 使用 msgq/AsyncQueue
    results: ResultStore,
    db: Db,
}

impl AsyncSqlExecutor {
    // 提交 SQL 任務，立即返回 job_id
    pub async fn execute(&self, sql: &str) -> Result<String, String>;
    
    // 輪詢結果
    pub async fn poll(&self, job_id: &str) -> Result<JobResult, String>;
    
    // 提交並等待結果
    pub async fn execute_and_wait(&self, sql: &str, timeout_ms: u64) -> Result<JobResult, String>;
    
    // 啟動背景 worker (tokio::spawn)
    pub fn spawn_workers(&self, count: usize);
}
```

#### Sync 模式 (基於 SyncQueue + blocking)

```rust
// SyncSqlExecutor - 阻塞 (基於 SyncQueue)
// 使用 SyncQueue，類似同步資料庫客戶端
pub struct SyncSqlExecutor {
    queue: SyncQueue,        // 使用 msgq/SyncQueue
    results: ResultStore,
    db: Db,
}

impl SyncSqlExecutor {
    // 同步執行 SQL，阻塞直到完成
    pub fn execute(&mut self, sql: &str) -> Result<JobResult>;
    
    // 同步執行並設定超時
    pub fn execute_with_timeout(&mut self, sql: &str, timeout_ms: u64) -> Result<JobResult>;
}
```

### 運作流程

```
Async 模式 (tokio):
┌──────────┐    ┌─────────┐    ┌────────┐    ┌─────────┐
│  Client  │───▶│ execute │───▶│ Async  │───▶│  tokio  │
│  (async) │    │ Job ID  │    │ Queue  │    │  spawn  │
└──────────┘    └─────────┘    └────────┘    └─────────┘
                                                 │
                                                 ▼
┌──────────┐    ┌─────────┐    ┌────────┐    ┌─────────┐
│  Client  │◀───│  poll   │◀───│Result  │◀──│  Store  │
│          │    │ Result  │    │ Store  │    │         │
└──────────┘    └─────────┘    └────────┘    └─────────┘

Sync 模式 (blocking):
┌──────────┐    ┌─────────┐    ┌────────┐
│  Client  │───▶│execute()│───▶│ Sync   │───▶ 結果直接返回
│  (sync)  │    │  Block  │    │ Queue  │
└──────────┘    └─────────┘    └────────┘
```

## 使用範例

### Async 模式 (tokio async)

```rust
use crate::msgq::sql::{AsyncSqlExecutor, ResultStore, JobResult};

let db = Db::new();
let results = ResultStore::Memory(Arc::new(RwLock::new(HashMap::new())));
let executor = AsyncSqlExecutor::new(db, results);

// 啟動 workers (tokio tasks)
executor.spawn_workers(4);

// 提交 SQL (非阻塞)
let job_id = executor.execute("SELECT * FROM users WHERE id = 1").await?;
println!("Job ID: {}", job_id);

// 輪詢結果
loop {
    match executor.poll(&job_id).await {
        Ok(JobResult::Select { columns, rows }) => {
            println!("結果: {} rows", rows.len());
            break;
        }
        Ok(JobResult::Error { message }) => {
            println!("錯誤: {}", message);
            break;
        }
        Err(_) => {
            tokio::time::sleep(Duration::from_millis(100)).await;
        }
    }
}
```

### Sync 模式 (blocking)

```rust
use crate::msgq::sql::SyncSqlExecutor;

let mut executor = SyncSqlExecutor::new(db, results);

// 阻塞直到結果
let result = executor.execute("INSERT INTO orders (user_id) VALUES (1)")?;

match result {
    JobResult::Insert { affected } => {
        println!("新增了 {} 筆", affected);
    }
    JobResult::Error { message } => {
        println!("錯誤: {}", message);
    }
    _ => {}
}
```

### Async + execute_and_wait (混合模式)

```rust
// 提交後等待結果，超時自動放棄
let result = executor
    .execute_and_wait("SELECT * FROM large_table", 5000)
    .await;

match result {
    Ok(JobResult::Select { rows, .. }) => println!("{} rows", rows.len()),
    Err(Timeout) => println!("查詢超時"),
    Err(Error) => println!("執行錯誤"),
}
```

### Batch Insert 範例 (tokio并行)

```rust
use futures::future::join_all;

// 使用 Async 模式提交大量插入
let mut handles = vec![];
for i in 0..1000 {
    let executor = executor.clone();
    let handle = tokio::spawn(async move {
        executor.execute(&format!("INSERT INTO logs (msg) VALUES ('log-{}')", i)).await
    });
    handles.push(handle);
}

// 等待所有任務完成
let results = join_all(handles).await;
let success_count = results.iter().filter(|r| r.is_ok()).count();
println!("成功提交: {}", success_count);
```

### 目錄結構


### Worker 實作 (基於 tokio)

```rust
// async_sql.rs
async fn worker_task(queue: AsyncQueue, results: ResultStore, db: Db) {
    loop {
        match queue.dequeue(0).await {
            Ok(Some(msg)) => {
                // 解碼 SQL 任務
                let (job_id, sql) = serde_json::from_slice(&msg.payload)
                    .map_err(|e| e.to_string())?;
                
                // 執行 SQL
                let result = db.execute(&sql).await;
                
                // 儲存結果
                results.store(&job_id, result);
                
                // 確認處理
                queue.ack(&msg.id).await;
            }
            Ok(None) => {
                tokio::time::sleep(Duration::from_millis(100)).await;
            }
            Err(e) => {
                eprintln!("Worker error: {}", e);
            }
        }
    }
}

// 啟動 workers
pub fn spawn_workers(&self, count: usize) {
    for _ in 0..count {
        let queue = self.queue.clone();
        let results = self.results.clone();
        let db = self.db.clone();
        
        tokio::spawn(async move {
            worker_task(queue, results, db).await;
        });
    }
}
```

## 技術總結

| 模式 | 底層實作 | 特性 |
|------|----------|------|
| AsyncSqlExecutor | AsyncQueue + tokio::spawn | 非阻塞、worker pool |
| SyncSqlExecutor | SyncQueue | 阻塞、簡單直接 |
| execute_and_wait | SyncQueue + tokio::timeout | 混合模式 |

此功能將 SQL 執行封裝為 Queue 服務，實現類似傳統 RDBMS 作為網路服務的運作模式，完全基於 tokio 和現有的 msgq/ 實作。
