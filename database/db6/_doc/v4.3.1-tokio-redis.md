# v4.3.1 PubSub 優化

## 規劃日期
2026-05-17

## 參考來源
mini-redis (tokio-rs) 原始碼分析

## 改進項目

### 1. Payload 類型優化

**現況**: 使用 `Vec<u8>`，每次複製
**改進**: 使用 `bytes::Bytes`，zero-copy

```rust
// Before
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct AsyncPubSubMessage {
    pub payload: Vec<u8>,
}

// After
use bytes::Bytes;

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct AsyncPubSubMessage {
    pub payload: Bytes,
}
```

**影響檔案**:
- `src/msgq/message.rs`
- `src/msgq/async_queue.rs`
- `src/msgq/async_pubsub.rs`

### 2. 加入 Background Purge Task

**現況**: 無後台任務處理訊息過期
**改進**: 參考 mini-redis，加入 purge task

```rust
use tokio::sync::Notify;
use std::collections::BTreeSet;
use tokio::time::{Duration, Instant};

struct Shared {
    state: Mutex<State>,
    background_task: Notify,
}

struct State {
    // ... existing fields ...
    expirations: BTreeSet<(Instant, String)>,  // 追蹤過期時間
}

async fn purge_expired_messages(shared: Arc<Shared>) {
    while !shared.is_shutdown() {
        if let Some(when) = shared.purge_expired_keys() {
            tokio::select! {
                _ = tokio::time::sleep_until(when) => {}
                _ = shared.background_task.notified() => {}
            }
        } else {
            shared.background_task.notified().await;
        }
    }
}
```

**說明**:
- 使用 BTreeSet 按時間排序，快速找到下一個過期的訊息
- 後台任務在背景執行，不阻塞主要操作

**影響檔案**:
- `src/msgq/async_queue.rs`

### 3. 同步機制優化

**現況**: 使用 `tokio::sync::RwLock`
**改進**: 考慮使用 `std::sync::Mutex`（critical section 小，無 await）

```rust
// Before
use tokio::sync::RwLock;
channels: Arc<RwLock<HashMap<...>>>,

// After - 適用於無 async 操作的 critical section
use std::sync::Mutex;
channels: Arc<Mutex<HashMap<...>>>,
```

**說明**:
- mini-redis 選擇 std::sync::Mutex 是因為 critical sections 很小
- 如果 critical section 會跨 await，才用 tokio::sync::RwLock

**影響檔案**:
- `src/msgq/async_pubsub.rs`

### 4. Broadcast Channel 容量參數化

**現況**: 硬編碼 1024
**改進**: 可設定容量

```rust
pub struct PubSubConfig {
    pub max_history: usize,
    pub history_enabled: bool,
    pub pattern_matching: bool,
    pub channel_capacity: usize,  // 新增
}

impl Default for PubSubConfig {
    fn default() -> Self {
        Self {
            max_history: 100,
            history_enabled: true,
            pattern_matching: true,
            channel_capacity: 1024,
        }
    }
}
```
