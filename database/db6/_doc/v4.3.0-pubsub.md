# v4.3 PubSub 強化

## 規劃日期
2026-05-17

## 功能清單

### Pattern Subscription

Glob 風格的主題匹配，訂閱者可以使用萬用字元訂閱多個主題。

```rust
// 訂閱 matching 的所有主題
pub fn subscribe_pattern(&mut self, pattern: &str, subscriber_id: &str) -> Result<()>;

// 消費 pattern 訂閱的訊息
pub fn consume_pattern(&mut self, pattern: &str, subscriber_id: &str) -> Result<Option<SyncPubSubMessage>>;
```

### Message History

新訂閱者可取得歷史訊息。

```rust
// 訂閱並取得最近 N 筆歷史訊息
pub fn subscribe_with_history(
    &mut self,
    channel: &str,
    subscriber_id: &str,
    history_count: usize,
) -> Result<Vec<SyncPubSubMessage>>;

// 取得歷史訊息
pub fn get_history(&self, channel: &str, count: usize) -> Result<Vec<SyncPubSubMessage>>;
```

### Wildcard Topics

複雜主題階層，支援多層萬用字元。

```rust
// Topic 格式: category.action.object
// 例如: user.created.admin, order.completed.customer

pub struct TopicMatcher {
    segments: Vec<Option<String>>,  // None = wildcard
}

impl TopicMatcher {
    pub fn new(pattern: &str) -> Self;
    pub fn matches(&self, topic: &str) -> bool;
}
```

### 配置選項

```rust
pub struct PubSubConfig {
    pub max_history: usize,         // 最大保留歷史訊息數
    pub history_enabled: bool,      // 啟用歷史訊息
    pub pattern_matching: bool,     // 啟用 pattern matching
}
```

## API 設計

### SyncPubSub 新增方法

```rust
impl SyncPubSub {
    // Pattern Subscription
    pub fn subscribe_pattern(&mut self, pattern: &str, subscriber_id: &str) -> Result<()>;
    pub fn consume_pattern(&mut self, pattern: &str, subscriber_id: &str) -> Result<Option<SyncPubSubMessage>>;

    // Message History
    pub fn subscribe_with_history(&mut self, channel: &str, subscriber_id: &str, history_count: usize) -> Result<Vec<SyncPubSubMessage>>;

}
```

## 實作細節

### Pattern Matching

Pattern 語法：
- `*` 匹配任意字元（不含 `.`）
- `**` 匹配任意階層
- `?` 匹配單一字元

範例：
- `news.*` 匹配 `news.sports`, `news.tech`
- `user.*.*` 匹配 `user.created.admin`, `user.deleted.system`
- `*.urgent` 匹配 `email.urgent`, `sms.urgent`

### History 儲存

```rust
struct ChannelHistory {
    channel: String,

    messages: Vec<SyncPubSubMessage>,
}
```

// "user.created.*" -> ["user", "created", None]
