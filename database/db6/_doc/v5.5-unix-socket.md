# v5.5 UNIX Domain Socket Transport

## Overview

Added UNIX Domain Socket (UDS) transport for db6, providing the highest performance path for local inter-process communication.

## Features

- Zero network overhead - direct socket access
- ~0.01ms latency (vs ~0.2ms for HTTP)
- Secure local-only access
- JSON request/response protocol
- Compatible with any language with Unix socket support

## API

Same unified API as HTTP, communicated via JSON over Unix socket.

### Socket Path

- Default: `/tmp/db6.sock`
- Configurable via `DB6_UDS_PATH` environment variable

### Request/Response Format

```json
// Request
{
  "method": "kv_put",
  "params": { "table_id": 1, "key": "user:1", "value": "{\"name\":\"John\"}" }
}

// Response
{ "status": "ok", "result": null }
```

### Methods

| Method | Params | Description |
|--------|--------|-------------|
| `kv_put` | table_id, key, value | Put key-value pair |
| `kv_get` | table_id, key | Get value by key |
| `kv_delete` | table_id, key | Delete key |
| `kv_scan` | table_id, start, end | Range scan |
| `kv_batch_put` | table_id, pairs | Batch write |
| `kv_range_delete` | table_id, start, end | Range delete |
| `sql_execute` | sql | Execute SQL |
| `fts_insert` | doc_id, text | Insert FTS document |
| `fts_search` | query | Full-text search |

## Usage

### Server

```bash
# Start with UDS enabled (default)
cargo run

# Custom socket path
DB6_UDS_PATH=/var/run/db6.sock cargo run
```

### Rust Client

```rust
use tokio::net::UnixSocket;
use tokio::io::{AsyncWriteExt, AsyncReadExt};

let socket = UnixSocket::new_stream()?;
let mut stream = socket.connect("/tmp/db6.sock").await?;

// Put
let req = serde_json::json!({
    "method": "kv_put",
    "params": { "table_id": 1, "key": "test", "value": "hello" }
});
stream.write_all(serde_json::to_string(&req)?.as_bytes()).await?;
stream.write_all(b"\n").await?;

// Read response
let mut buf = String::new();
stream.read_line(&mut buf).await?;
let resp: serde_json::Value = serde_json::from_str(&buf)?;
```

### Python Client

```python
import socket
import json

sock = socket.socket(socket.AF_UNIX, socket.SOCK_STREAM)
sock.connect("/tmp/db6.sock")

# Put
req = {"method": "kv_put", "params": {"table_id": 1, "key": "test", "value": "hello"}}
sock.sendall((json.dumps(req) + "\n").encode())

# Get
req = {"method": "kv_get", "params": {"table_id": 1, "key": "test"}}
sock.sendall((json.dumps(req) + "\n").encode())

resp = sock.recv(4096)
print(json.loads(resp))
```

### Node.js Client

```javascript
import { createConnection } from 'node:net';

const sock = createConnection({ path: '/tmp/db6.sock' });

// Put
sock.write(JSON.stringify({
    method: 'kv_put',
    params: { table_id: 1, key: 'test', value: 'hello' }
}) + '\n');

// Get
sock.write(JSON.stringify({
    method: 'kv_get',
    params: { table_id: 1, key: 'test' }
}) + '\n');

sock.on('data', (data) => {
    console.log(JSON.parse(data.toString()));
});
```

## Implementation

### File Structure

```
src/
├── server/
│   ├── mod.rs         # Module exports
│   ├── server.rs      # Multi-transport server (spawns UDS)
│   └── uds.rs         # UNIX Domain Socket handler
```

### Server Handler (`src/server/uds.rs`)

```rust
use tokio::net::{UnixSocket, UnixListener};
use tokio::io::{AsyncReadExt, AsyncWriteExt};

pub async fn serve_uds(path: &Path) -> Result<()> {
    let socket = UnixSocket::new_stream()?;
    socket.bind(path)?;
    let listener = socket.listen(1024)?;

    loop {
        let (stream, _) = listener.accept().await?;
        tokio::spawn(async move {
            handle_connection(stream).await;
        });
    }
}

async fn handle_connection(stream: UnixStream) {
    // Read JSON request, write JSON response
    // Protocol: one JSON line per request/response
}
```

### Protocol

1. Client connects to `/tmp/db6.sock`
2. Client sends JSON request followed by `\n`
3. Server processes request via unified handler
4. Server sends JSON response followed by `\n`
5. Connection remains open for subsequent requests

## Configuration

| Env Variable | Default | Description |
|--------------|---------|-------------|
| `DB6_UDS_PATH` | `/tmp/db6.sock` | Unix socket path |
| `DB6_UDS_ENABLED` | `true` | Enable UDS transport |

## Dependencies

```toml
tokio = { version = "1", features = ["net", "rt-multi-thread", "macros"] }
serde = { version = "1", features = ["derive"] }
serde_json = "1"
```

> Note: `UnixSocket` is built into tokio with the `net` feature. No extra crate needed.

## Performance Comparison

| Transport | Latency | Throughput | Use Case |
|-----------|---------|-------------|----------|
| **UDS** | ~0.01ms | Highest | Local, latency-critical |
| gRPC | ~0.1ms | High | Remote, multi-language |
| HTTP | ~0.2ms | Medium | Web, simple clients |
| WebSocket | ~0.2ms | Medium | Real-time, streaming |

## Notes

- Socket file is deleted on server shutdown
- Permissions: 0600 (owner read/write only)
- No authentication needed (local-only access)
- Consider `SO_REUSEADDR` for quick restart
