# v3.14 - 視圖 (View) 支援

## 日期：2026-05-17

## 目標

在 SQL Executor 模組中，完整支援視圖系統（Views），包含 `CREATE VIEW`、`DROP VIEW` 以及對視圖的動態查詢（View Querying）。

## 已實作

### 1. AST 結構與解析支援
透過系統已有的解析器，成功解析視圖相應的 SQL 語句，其對應的 AST 結構如下：
```rust
CreateViewStmt {
    if_not_exists: bool,
    name: String,
    query: Box<SelectStmt>, // 視圖底層實際的查詢邏輯
}

DropViewStmt {
    if_exists: bool,
    name: String,
}
```

### 2. Executor 內部狀態擴充
在原先的 `Executor` 與 `SqlExecutor` 執行實例中，新增了 `views` 這個屬性，用於實時快取已建立的視圖查詢。
```rust
pub struct Executor {
    engine: Box<dyn StorageEngine>,
    views: std::collections::HashMap<String, crate::sql::parser::ast::SelectStmt>,
}
```

### 3. 操作指令開發
已實作處理不同 AST 語句的分支函式：
- **`execute_create_view`**：將 `CREATE VIEW` 所封裝的查詢取出並保存於 Executor 狀態內的雜湊表（HashMap）中。
- **`execute_drop_view`**：處理 `DROP VIEW` 指令，將特定視圖從內部雜湊表中移除。支援加上 `IF EXISTS` 保護。

### 4. 視圖遞迴執行（Recursive Resolution）
修改了 `execute_select`，使系統在從 `FROM` 抓取資料來源時，能優先從自身的 `views` 雜湊表中檢查該名稱是否屬於現有視圖。
如果找到指定的視圖，便會複製視圖的 `SelectStmt` 並且**遞迴呼叫** `execute_select`，獲取視圖底層帶來的結果後，再繼續套用當前語句外部的條件封裝：
```rust
if let Some(view_query) = self.views.get(&main_table).cloned() {
    return self.execute_select(&view_query);
}
```
這樣的實作手法，使視圖可以天然支援未來和過去所有的子查詢、`JOIN`、`GROUP BY` 與 `JSON Path` 操作等。

## 測試案例示範

您可以在程式碼內直接透過 `Executor` 嘗試以下指令驗證：

```sql
-- 1. 準備基礎資料
INSERT INTO users VALUES ('{"name":"Alice","role":"admin"}');
INSERT INTO users VALUES ('{"name":"Bob","role":"member"}');

-- 2. 建立視圖 (僅過濾特定角色)
CREATE VIEW admin_users AS SELECT * FROM users WHERE @.role = 'admin';

-- 3. 查詢視圖
SELECT @.name FROM admin_users; 
-- 預期僅會回傳 Alice

-- 4. 刪除視圖
DROP VIEW IF EXISTS admin_users;
```

## 現有挑戰與未來優化

1. **視圖持久化 (Persistence) 尚未實作**
   當前 `views` 之資訊僅單純保存在記憶體（`HashMap`）內。如果使用提供 Disk 持久化能力的資料儲存庫，Executor 若重新初始化，視圖設定將會喪失。未來應將視圖建立語法轉存進隱含性的 Metadata系統表 之中。
2. **無窮遞迴防範 (Cyclic Reference Defense)**
   由於採遞迴調用 (`execute_select(&view_query)`) 去獲取底層資料，尚未內建防止「`VIEW A 依賴 VIEW B` 且 `VIEW B 依賴 VIEW A`」的情境。目前此舉將導致執行期呼叫堆疊溢出 (Stack Overflow)。
3. **寫入視圖 (View Insertion/Update)**
   受限於時間，目前無法針對簡單視圖直接進行 `INSERT INTO view ...`，對視圖操作一律僅供讀取。
