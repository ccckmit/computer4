# 關聯式資料庫（Relational Database）

## 概述

關聯式資料庫（Relational Database）是一種基於 E.F. Codd 於 1970 年提出的關聯模型（Relational Model）的資料庫管理系統。關聯式資料庫使用二維表格（稱為「關聯」或「關係」）來組織資料，表格之間通過鍵（Key）建立關聯。

關聯式資料庫的核心優勢在於：
- **資料完整性**：通過約束、鍵和交易機制確保資料正確
- **SQL 標準化**：統一的查詢語言，適合複雜查詢
- **交易支援**：完整的 ACID 保證
- **成熟的生態系統**：數十年的發展，工具豐富

## 關聯模型的基礎

### 表格（Table / Relation）

關聯式資料庫的基本結構是表格，由行和列組成：

| id (主鍵) | name | email | age |
|-----------|------|-------|-----|
| 1 | 張三 | zhang@example.com | 25 |
| 2 | 李四 | li@example.com | 30 |

### 術語對照

| SQL 術語 | 數學術語 | 說明 |
|----------|----------|------|
| 表（Table） | 關係（Relation） | 資料的集合 |
| 行（Row） | 元組（Tuple） | 一筆記錄 |
| 列（Column） | 屬性（Attribute） | 一個欄位 |
| 主鍵（Primary Key） | 候選鍵（Candidate Key） | 唯一識別每一行 |
| 外鍵（Foreign Key） | 參照鍵（Reference Key） | 關聯另一表 |

### 鍵（Key）

**主鍵（Primary Key）**：唯一識別表中每一行的欄位或欄位組合。一個表只有一個主鍵。

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL
);
```

**外鍵（Foreign Key）**：一個或多個欄位，引用另一表的主鍵，用於建立關聯：

```sql
CREATE TABLE orders (
    id INTEGER PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    total DECIMAL(10,2)
);
```

**唯一鍵（Unique Key）**：確保某欄位或欄位組合的值唯一，但不像主鍵那樣要求非空。

## 正規化（Normalization）

正規化是設計關聯式資料庫的重要原則，目的是減少資料冗餘和更新異常。

### 第一正規化（1NF）

去除重複的欄位群，確保每個儲存格都是原子值（不可再分）：

```sql
-- 不符合 1NF：address 不是原子值
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    name TEXT,
    address TEXT  -- "台北市信義區,101"
);

-- 符合 1NF：地址被拆分
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    name TEXT,
    city TEXT,
    district TEXT,
    postal_code TEXT
);
```

### 第二正規化（2NF）

在滿足 1NF 的基礎上，所有非主鍵欄位完全依賴主鍵（消除部分函數依賴）：

```sql
-- 不符合 2NF：course_name 只依賴 course_id，不完全依賴 (student_id, course_id)
CREATE TABLE enrollments (
    student_id INTEGER,
    course_id INTEGER,
    course_name TEXT,  -- 只依賴 course_id
    grade TEXT,
    PRIMARY KEY (student_id, course_id)
);

-- 符合 2NF：分離到獨立的 courses 表
CREATE TABLE enrollments (
    student_id INTEGER,
    course_id INTEGER,
    grade TEXT,
    PRIMARY KEY (student_id, course_id),
    FOREIGN KEY (course_id) REFERENCES courses(id)
);
```

### 第三正規化（3NF）

在滿足 2NF 的基礎上，消除遞移依賴（非主鍵欄位之間的依賴）：

```sql
-- 不符合 3NF：department 依賴於 location_id，而 location_id 依賴於 department_id
CREATE TABLE departments (
    department_id INTEGER PRIMARY KEY,
    department_name TEXT,
    location_id INTEGER,
    location_name TEXT  -- 遞移依賴
);

-- 符合 3NF：分離到 locations 表
CREATE TABLE departments (
    department_id INTEGER PRIMARY KEY,
    department_name TEXT,
    location_id INTEGER REFERENCES locations(id)
);
```

### Boyce-Codd 正規化（BCNF）

第三正規化的一個更嚴格版本，處理主鍵重疊的情況（較少見）。

## 交易（Transaction）

關聯式資料庫的交易機制提供完整的 ACID 保證：

```sql
BEGIN TRANSACTION;

UPDATE accounts SET balance = balance - 1000 WHERE id = 1;
UPDATE accounts SET balance = balance + 1000 WHERE id = 2;

-- 如果兩條語句都成功，提交
COMMIT;

-- 如果任何一條失敗，回滾所有更改
ROLLBACK;
```

詳細說明請參閱 [ACID.md](ACID.md) 和 [Transaction.md](Transaction.md)。

## JOIN（連接）

JOIN 是關聯式資料庫的核心操作，用於合併多個表的資料：

### 語法

```sql
SELECT users.name, orders.total
FROM users
INNER JOIN orders ON users.id = orders.user_id
WHERE orders.date >= '2024-01-01';
```

### 類型

| JOIN 類型 | 說明 | 結果 |
|-----------|------|------|
| INNER JOIN | 只返回兩表都有匹配的記錄 | 交集 |
| LEFT JOIN | 返回左表所有記錄，無匹配時以 NULL 填充 | 左表全集 |
| RIGHT JOIN | 返回右表所有記錄 | 右表全集 |
| FULL OUTER JOIN | 返回兩表的所有記錄 | 聯集 |
| CROSS JOIN | 笛卡爾積 | 所有組合 |

### 執行方式

查詢最佳化器會選擇最有效的 JOIN 演算法：

- **Nested Loop Join**：雙重迴圈，適合小表
- **Hash Join**：建立哈希表，適合大表等值連接
- **Sort-Merge Join**：先排序再合併，適合有序輸入

## 索引（Index）

索引是提升查詢效能的關鍵結構：

```sql
-- 單欄位索引
CREATE INDEX idx_users_email ON users(email);

-- 複合索引
CREATE INDEX idx_orders_user_date ON orders(user_id, date);

-- 唯一索引
CREATE UNIQUE INDEX idx_users_email_unique ON users(email);
```

詳細說明請參閱 [Index.md](Index.md)。

## 知名關聯式資料庫

### PostgreSQL

最先進的開源關聯式資料庫：
- 完全支援 SQL 標準
- 豐富的資料類型（JSON、陣列、範圍等）
- 強大的擴展機制（PostGIS、pgvector 等）
- 活躍的社群

### MySQL

熱門的開源關聯式資料庫：
- LAMP 堆疊的核心元件
- 簡單易用，效能優異
- InnoDB 引擎支援交易和行級鎖

### SQLite

輕量級嵌入式資料庫：
- 無需伺服器程序
- 單檔案儲存
- 廣泛用於行動應用和嵌入式系統

### Oracle Database

企業級商業資料庫：
- 功能最全面的資料庫系統
- 成熟的優化器和工具鏈
- 昂貴但穩定

### SQL Server

微軟的企業級資料庫：
- 與 Azure 深度整合
- 強大的商業智慧功能
- T-SQL 作為查詢語言

## 在 db6 中的對應

db6 的 SQL 層是對關聯式資料庫概念的一個輕量級實現：

| 關聯式概念 | db6 實現 |
|-----------|----------|
| 表格 | StorageEngine 的 table_id |
| 列 | 應用層定義（如 JSON 格式） |
| 主鍵 | 應用層約束 |
| SQL 引擎 | src/sql/parser, planner, executor |
| 索引 | 可在 KV 層之上構建 |

db6 的 [BTreeEngine](../src/engine/btree/) 最適合關聯式工作負載，因為 B+Tree 提供了良好的讀取效能和完整的交易支援。
