# v0.3 — 純 Rust 邏輯綜合器（精簡版）

## 目標

以純 Rust 實作 Verilog-2005 子集的**邏輯綜合器**，輸出 iCE40 可對應的 JSON 網表格式，逐步取代 Yosys。

## 各模組詳情

### parser.rs — Verilog Tokenizer + Parser

- Tokenizer：`tokenize(src: &str) -> Vec<Token>`
  - 支援關鍵字：`module, endmodule, input, output, wire, reg, assign, always, posedge, negedge, if, else, begin, end`
  - 支援數字：`4'b1010`, `8'hFF`, `32'd100`
  - 支援運算子：`& | ^ ~ + - * / % << >> {} [] ? :`
- Parser (Recursive Descent)：`parse(tokens) -> AstModule`
  - Module → Port list → Body (assign, always, wire, submodule)
  - 每個 `AstModule` 包含 inputs, outputs, wires, assigns, always_blocks, instances

### elab.rs — 展開與例化

- 展開單一模組為扁平網表
- 處理參數化匯流排（`wire [7:0] bus` → 8 條 net）
- 解析 `always @(posedge clk)` 推導 flip-flop

### netlist.rs — 內部網表

為 `v2f-core` 的 `Netlist` 提供建構函式：

```rust
pub struct Netlist {
    pub cells: Vec<Cell>,
    pub nets: Vec<Net>,
    pub top: String,
}

pub struct Cell {
    pub kind: CellKind,     // LUT, DFF, CARRY, IO, BRAM
    pub inputs: Vec<(String, NetRef)>,
    pub outputs: Vec<(String, NetRef)>,
    pub params: HashMap<String, Value>,
}
```

### techmap.rs — iCE40 技術映射

將通用閘極 (`AND`, `OR`, `XOR`, `MUX`, `DFF`) 對應到 iCE40 的實體單元：

| 邏輯 | iCE40 對應 |
|------|-----------|
| combinational logic | `ICESTORM_LC` (LUT4 + DFF + Carry) |
| flip-flop | `ICESTORM_LC` (.DFF_ENABLE=1) |
| arithmetic | `ICESTORM_LC` (.CARRY_ENABLE=1) |
| I/O pad | `SB_IO` / `SB_GB_IO` |

LUT4 對應演算法：

```
func_to_lut4(truth_table: [bool; 16], inputs: [Net; 4])
  → ICESTORM_LC { .LUT_INIT = truth_table, .DFF_ENABLE = 0, ... }
```

### synth.rs — 綜合主流程

```
理順：
  1. parse   → AstModule
  2. elab    → FlatNetlist (logic gates, DFFs)
  3. opt     → 常數傳遞、冗餘消除
  4. techmap → MappedNetlist (ICESTORM_LC...)
  5. output  → JSON (相容 nextpnr 格式)
```

JSON 輸出格式（與 Yosys `synth_ice40 -json` 相容）：

```json
{
  "creator": "v2f-synth v0.3",
  "modules": {
    "top": {
      "ports": {
        "clk": { "direction": "input", "bits": [1] },
        "led": { "direction": "output", "bits": [2] }
      },
      "cells": {
        "$1": { "type": "ICESTORM_LC", ... }
      },
      "netnames": {
        "led": { "bits": [2], "hide_name": 0 }
      }
    }
  }
}
```

## 驗收標準

1. `cargo build` 通過
2. 可綜合以下電路並與 Yosys 輸出比對（功能對等，不要求 binary 一致）：
   - 單一組合邏輯：`assign out = a & b | c`
   - DFF 暫存器：`always @(posedge clk) q <= d`
   - 計數器：`always @(posedge clk) cnt <= cnt + 1`
   - 多 bit 運算：`assign y = a + b`
3. `v2f build --pure-synth blinky.v` 可不經 Yosys 完成綜合
4. **不支援** 的語法要有明確錯誤訊息
