# v0.1 — CLI 框架與外部工具整合

## 目標

建立 CLI 工具 `v2f`，以 subprocess 呼叫既有開源工具（Yosys / nextpnr / icepack / openFPGALoader），完成 Lattice iCE40 HX8K 的完整 E2E 流程。

## 建立方式

於 `eda/verilog2fpga/` 下建立 Cargo workspace：

```
verilog2fpga/
├── Cargo.toml              # [workspace] members = ["v2f-cli", "v2f-core"]
├── v2f-cli/
│   ├── Cargo.toml          # 依賴 v2f-core, clap, subprocess
│   └── src/
│       ├── main.rs
│       ├── synth.rs        # Yosys subprocess 包裝
│       ├── pnr.rs          # nextpnr subprocess 包裝
│       ├── pack.rs         # icepack subprocess 包裝
│       └── prog.rs         # openFPGALoader subprocess 包裝
├── v2f-core/
│   ├── Cargo.toml
│   └── src/
│       ├── lib.rs          # pub mod device, config, error
│       ├── device.rs       # Device enum (HX1K, HX8K, UP5K…)
│       ├── config.rs       # Config struct (TOML 載入)
│       └── error.rs        # 統一錯誤型別
└── _doc/
    ├── plan.md
    └── v0.1.md
```

## CLI 介面

```
v2f build input.v                      # 自動偵測裝置（預設 hx8k）
v2f build input.v --device hx1k
v2f build input.v --device up5k --pcf pins.pcf

v2f synth [options] input.v
v2f pnr [options] netlist.json
v2f pack [options] input.asc
v2f prog [options] bitstream.bin

v2f list-devices                       # 列出支援的裝置
v2f check                              # 檢查外部工具是否存在
```

## 子命令實作細節

### synth.rs — Yosys 包裝

```bash
yosys -p "synth_ice40 -json <output.json> -top <top_module>" -q <input.v>
```

- 先於 PATH 搜尋 `yosys`
- 支援 `-top` (預設探測第一個 module)
- 回傳 JSON 檔案路徑

### pnr.rs — nextpnr 包裝

```bash
nextpnr-ice40 --<device> --json <input.json> --pcf <constraints.pcf> --asc <output.asc>
```

- `--device` 對應 Device enum → CLI flag
- `--pcf` 可選
- 回傳 ASC 檔案路徑

### pack.rs — icepack 包裝

```bash
icepack <input.asc> <output.bin>
```

- 回傳 bin 檔案路徑

### prog.rs — openFPGALoader 包裝

```bash
openFPGALoader -b ice40_generic <bitstream.bin>
```

- 支援 `--cable` 參數指定下載器

## 驗收標準

1. `cargo build` 通過
2. `v2f check` 正確回報各工具是否可用
3. 用 iCE40-HX8K 開發板 + 一個 LED 閃爍範例，執行完整流程：
   - `v2f build blinky.v`
   - 成功產生 `blinky.bin`
4. `v2f list-devices` 輸出 HX1K / HX4K / HX8K / LP1K / UP5K
