# v0.4 — 純 Rust 佈局佈線（精簡版）

## 目標

以純 Rust 實作 Lattice iCE40 專用的**放置與繞線**引擎，輸出適用於 `v2f-bitstream` 的 ASC 格式。

## 放置引擎 (`place.rs`)

### 演算法：Simulated Annealing

```rust
pub fn place(netlist: &Netlist, device: &Device) -> Placement {
    // 1. 初始化：隨機放置（合法位置）
    // 2. 溫度迴圈 T = T_start → T_end
    //    a. 挑選一個 cell，嘗試移至相鄰 tile
    //    b. ΔC = 移動後代價 - 移動前代價
    //    c. if ΔC < 0 or random() < exp(-ΔC / T) → 接受
    // 3. 降溫 T *= cooling_rate
    // 4. 回傳 Placement { cell → (row, col) 對應 }
}
```

### 代價函數

```
cost = w_wire * wire_length + w_cong * congestion + w_timing * timing_slack
```

- `wire_length`：半周長 (HPWL) — bounding box 半周長總和
- `congestion`：各 tile 繞線資源使用率超標懲罰
- `timing_slack`：簡化版時序估算（組合深度）

### iCE40 合法放置位置

| Cell 類型 | 可放置 Tile |
|-----------|------------|
| ICESTORM_LC (logic) | Logic tile (row 1..N-1, col 0..M-1) |
| SB_IO | IO tile (row 0 或 row N-1) |
| SB_GB_IO | IO tile (特定 col) |
| SB_RAM40_4K | BRAM tile (特定 row) |

### 演算法：PathFinder (negotiated congestion)

```rust
    // 第一遍：忽略擁擠，找最短路徑

}
```

### 繞線資源圖 (`arch.rs`)

iCE40 連線結構抽象：

```
Logic Tile:
  ┌──────────────┐
  │  LUT → FF     │
  │  ↓    ↓       │
  │  Routing MUX  │──→ 水平/垂直 track
  │  ↑    ↑       │
  │  Local Wire   │
  └──────────────┘
```

資源圖節點類型：

- `PinNode(cell, port)` — 單元輸入/輸出腳
- `TileInput(tile, track)` — tile 輸入埠
- `TileOutput(tile, track)` — tile 輸出埠
- `WireSegment(tile, track, len)` — 線段
- `SwitchNode(tile, track_in, track_out)` — 可程式切換點

### 目前簡化

- 僅處理 **logic tile** 間繞線（IO/BRAM 固定放置）
- 使用 **A\*** 搭配曼哈頓距離啟發式
- 支援單一軌道長度（wiring length = 2/4/8，iCE40 典型值）

## ASC 輸出 (`asc_out.rs`)

將 `Placement` + `Routing` 轉為 ASC 格式：

```
.logic_tile <row> <col>
  .lut <out_bit> <in0> <in1> <in2> <in3> "<lut_init>"
  .ff <out_bit> ce=<ce_net> sr=<sr_net> ...
  .carry <bit> ci=<ci_net> ...
  .wiring <switch_point> <net_id>

.io_tile <row> <col>
  .pad <pad_name> <direction> ...
```

## 協同運作

```
v2f build --pure-pnr blinky.v
  → [v2f-synth] output.json
  → [v2f-pnr]   output.asc
  → [v2f-bitstream] output.bin
```

`v2f build` 在 v0.4 支援三種模式：

| 模式 | 綜合 | P&R | 位元流 | 需外部工具 |
|------|------|-----|--------|-----------|
| `--yosys` | Yosys | nextpnr | v2f-bitstream | Yosys + nextpnr |
| `--pnr-only` | Yosys | v2f-pnr | v2f-bitstream | Yosys |
| `--pure-rust` | v2f-synth | v2f-pnr | v2f-bitstream | 無 |

## 驗收標準

1. `cargo build` 通過
2. 能對小型電路（< 100 LUT）成功 P&R：
   - 8-bit 計數器
   - 4-bit 加法器 + 暫存器
3. 輸出 ASC 能被 `v2f-bitstream` 正確打包
4. 功能正確性：與 nextpnr 輸出比對結果一致（可燒錄至 FPGA 驗證）
5. 支援 HX1K、HX8K 兩種裝置
