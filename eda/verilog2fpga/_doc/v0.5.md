# v0.5 — 純 Rust 燒錄器與 v2f-rust 橋接

## 目標

1. 以純 Rust 實作 FTDI/JTAG 驅動，直接對 iCE40 FPGA 進行燒錄，無需 openFPGALoader
2. 建立 `v2f-rust` 橋接，讓 Rust 內嵌的 HDL（rhdl 風格）可直接輸出至本工具鏈

## v2f-programmer：燒錄驅動

### FTDI 通訊層 (`ftdi.rs`)

```rust
pub struct FtdiDevice {
    handle: ftdi::Device,
}

impl FtdiDevice {
    pub fn open(vid: u16, pid: u16, interface: Interface) -> Result<Self>;
    pub fn write(&mut self, data: &[u8]) -> Result<()>;
    pub fn read(&mut self, len: usize) -> Result<Vec<u8>>;
}
```

支援兩種 backend：
- `libftd2xx-sys` — FTDI 官方 D2XX 驅動（高效能）
- `ftdi-rs` — 純 Rust libftdi wrapper（跨平台）

### JTAG 狀態機 (`jtag.rs`)

標準 JTAG 的 **Test Access Port (TAP)** 控制器，狀態圖：

```
Test-Logic-Reset
    ↓
Run-Test/Idle
    ↓
Select-DR-Scan ─→ Select-IR-Scan
    ↓                  ↓
Capture-DR        Capture-IR
    ↓                  ↓
Shift-DR ──→ ...  Shift-IR
    ↓                  ↓
Update-DR         Update-IR
```

實作 `JtagStateMachine`：

```rust
pub struct JtagStateMachine<D: FtdiIo> {
    tms: Vec<bool>,    // Test Mode Select 序列
    tdi: Vec<u8>,      // Test Data In
    tdo: Vec<u8>,      // Test Data Out
    state: TapState,
}
```

### iCE40 程式化流程 (`ice40_prog.rs`)

```
iCE40 SPI Flash 燒錄 (SPI Slave Mode)：
  1. 進入 CRAM 直接配置模式
  2. 寫入 CRAM (透過 JTAG 或 SPI)
  3. 喚醒 FPGA (WAKEUP 指令)

iCE40 SRAM 直接載入 (`ice40_cfg.rs`)：
  1. Reset CRAM
  2. Send 8 dummy bytes (0x00)
  3. Send bitstream payload
  4. Send CRC
  5. Send 48 dummy clocks
  6. Wakeup
```

### CLI 整合

```sh
v2f prog bitstream.bin                     # 自動偵測燒錄器
v2f prog bitstream.bin --driver jtag       # JTAG FT2232H
v2f prog bitstream.bin --driver spi        # SPI (ice40_dev_board)
v2f prog bitstream.bin --driver openfpga   # 呼叫 openFPGALoader
```

## v2f-rust：Rust HDL 橋接

### 語法 (macro 風格)

受 `verilog2rust` 的 rhdl 啟發，但直接輸出網表給本工具鏈：

```rust
use v2f_rust::fpga;

fpga! {
    module Blinky (
        input  clk: 1,
        output led: 1
    ) {
        let counter = Reg::<26>(0);    // 26-bit counter
        counter.next = counter + 1;
        led = counter[25];
    }
}
```

### compile.rs — Rust HDL → Netlist

```rust
pub fn compile_hdl<TokenStream>(tokens: TokenStream) -> Netlist
```

- 使用 `proc-macro2` + `syn` 解析 Rust token
- 支援：`Reg<T>`、`Wire<T>`、運算重載（`+`, `&`, `|`, `^`, `~`）
- `if` / `match` 對應組合/循序邏輯
- 輸出 `v2f_core::netlist::Netlist`

### backend.rs — 輸出後端

```rust
pub fn to_verilog(netlist: &Netlist) -> String       // 輸出 .v
pub fn to_json(netlist: &Netlist) -> String           // 輸出 Yosys 相容 JSON
pub fn to_synth(netlist: &Netlist) -> Netlist          // 直接送 v2f-synth
```

### CLI 整合

```sh
v2f build --lang rust src/main.rs            # Rust HDL 專案
v2f build --lang verilog blinky.v            # Verilog (既有)
```

`--lang rust` 時：
1. `cargo expand` 展開 proc macro
2. `compile_hdl` → Netlist
3. JSON 輸出 → `v2f-synth`（略過 Verilog parser，直接從 netlist 開始）
4. → `v2f-pnr` → `v2f-bitstream`

## 驗收標準 (v0.5)

1. `cargo build` 通過（所有 crate）
2. FTDI 驅動可連接 FT2232H 開發板
3. 燒錄 iCE40 bitstream 至實體 FPGA（LED blinky 範例）
4. Rust HDL macro 可編譯並輸出正確網表
5. `v2f build --lang rust` 完整 E2E（至 bitstream）
6. 無任何外部工具依賴：`v2f check` 回報「所有 Rust-native 模組已就緒」
